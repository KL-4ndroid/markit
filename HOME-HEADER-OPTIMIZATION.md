# 首頁 Header 按鈕優化

## 📋 修改內容

### 移除的按鈕
- ❌ 「離線模式」文字標籤
- ❌ 「🔍」調試按鈕

### 新增的按鈕

#### 1. 同步狀態按鈕
- **位置**：Header 右上角第一個按鈕
- **樣式**：圓形按鈕，白色半透明背景
- **圖示**：根據同步狀態動態變化
  - 🔄 `Loader2`（旋轉動畫）- 同步中
  - ✅ `CheckCircle` - 已同步
  - ⚠️ `AlertCircle` - 同步失敗
  - ☁️ `Cloud` - 閒置
  - 📴 `CloudOff` - 離線/未登入
- **功能**：點擊跳轉到設定頁面
- **Tooltip**：顯示當前狀態（已同步/離線模式/未登入）

#### 2. 使用者登入狀態按鈕
- **位置**：Header 右上角第二個按鈕
- **樣式**：圓形按鈕，白色半透明背景
- **圖示**：根據登入狀態變化
  - 👤 `UserCircle` - 已登入
  - 👤 `User` - 未登入
- **功能**：點擊跳轉到設定頁面
- **Tooltip**：
  - 已登入：顯示用戶 email
  - 未登入：顯示「未登入」

## 💡 設計理念

### 1. 簡潔優先
- 移除文字標籤，只保留圖示
- 減少視覺噪音，更加簡潔

### 2. 狀態可視化
- 同步狀態一目了然（圖示顏色和動畫）
- 登入狀態清晰明確（不同圖示）

### 3. 一致性
- 兩個按鈕樣式統一
- 都使用圓形按鈕 + 半透明背景
- 都有 hover 效果

### 4. 可訪問性
- 保留 `title` 屬性（滑鼠懸停顯示說明）
- 點擊跳轉到設定頁面（統一入口）

## 🔧 技術實現

### 同步狀態邏輯

```typescript
const getSyncIcon = () => {
  if (!user || !isConfigured) {
    return <CloudOff className="w-5 h-5" />;
  }
  
  switch (status) {
    case SyncStatusEnum.SYNCING:
      return <Loader2 className="w-5 h-5 animate-spin" />;
    case SyncStatusEnum.SUCCESS:
      return <CheckCircle className="w-5 h-5" />;
    case SyncStatusEnum.ERROR:
      return <AlertCircle className="w-5 h-5" />;
    case SyncStatusEnum.OFFLINE:
      return <CloudOff className="w-5 h-5" />;
    default:
      return <Cloud className="w-5 h-5" />;
  }
};
```

### 用戶狀態邏輯

```typescript
const getUserIcon = () => {
  if (user) {
    return <UserCircle className="w-5 h-5" />;  // 已登入
  }
  return <User className="w-5 h-5" />;  // 未登入
};
```
