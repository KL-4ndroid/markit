# 員工邀請系統 - 限制一個帳號只能加入一個團隊

## 🔒 業務邏輯限制

為了避免數據混亂和權限衝突，系統限制：**一個帳號只能加入一個團隊**。

---

## ✅ 實作內容

### 1. **資料庫層檢查**

**檔案**: `supabase/migrations/028_staff_invitations.sql`

在 `accept_invitation_and_bind` 函數中添加檢查：

```sql
-- ✅ 檢查用戶是否已經加入其他團隊
SELECT COUNT(*)
INTO v_existing_owner_count
FROM staff_relationships
WHERE staff_id = p_staff_id
AND status IN ('pending', 'active');

IF v_existing_owner_count > 0 THEN
  RETURN QUERY SELECT FALSE, '您已經是其他老闆的員工，一個帳號只能加入一個團隊'::TEXT, NULL::UUID;
  RETURN;
END IF;
```

### 2. **前端 UI 提示**

**檔案**: `app/join/page.tsx`

在邀請頁面添加明確的提示：

```
┌─────────────────────────────────────┐
│  ⚠️ 重要提示                         │
│  一個帳號只能加入一個團隊。          │
│  如果您已經是其他老闆的員工，        │
│  將無法加入此團隊。                  │
└─────────────────────────────────────┘
```

## 🔄 完整流程

### 情境 1：用戶未加入任何團隊

1. 用戶點擊邀請連結
2. 驗證 Token 成功
3. 檢查用戶是否已加入其他團隊 → **否**
4. 允許加入團隊 ✅

### 情境 2：用戶已加入其他團隊

1. 用戶點擊邀請連結
2. 驗證 Token 成功
3. 檢查用戶是否已加入其他團隊 → **是**
4. 拒絕加入，顯示錯誤訊息：「您已經是其他老闆的員工，一個帳號只能加入一個團隊」❌

### 情境 3：用戶已是此老闆的員工

1. 用戶點擊邀請連結
2. 驗證 Token 成功
3. 檢查是否已是此老闆的員工 → **是**
4. 拒絕加入，顯示錯誤訊息：「您已經是此老闆的員工」❌

---

## 🧪 測試步驟

### 測試 1：正常加入團隊

1. 用戶 A（未加入任何團隊）
2. 點擊老闆 B 的邀請連結
3. 確認加入
4. **應該成功加入** ✅

### 測試 2：嘗試加入第二個團隊

1. 用戶 A（已是老闆 B 的員工）
2. 點擊老闆 C 的邀請連結
3. 確認加入
4. **應該顯示錯誤**：「您已經是其他老闆的員工，一個帳號只能加入一個團隊」❌

### 測試 3：重複加入同一個團隊

1. 用戶 A（已是老闆 B 的員工）
2. 點擊老闆 B 的邀請連結
3. 確認加入
4. **應該顯示錯誤**：「您已經是此老闆的員工」❌

## 📝 錯誤訊息

| 情境 | 錯誤訊息 |
|------|---------|
| 已加入其他團隊 | 您已經是其他老闆的員工，一個帳號只能加入一個團隊 |
| 已是此老闆的員工 | 您已經是此老闆的員工 |
| Token 不存在 | 邀請連結不存在 |
| Token 已過期 | 邀請連結已過期 |

## 🔧 如何離開團隊

如果用戶想要離開當前團隊並加入新團隊，需要：

1. 聯繫當前老闆，請求移除員工關係
2. 老闆在 `/settings` → 員工管理 → 移除員工
3. 用戶才能加入新團隊
