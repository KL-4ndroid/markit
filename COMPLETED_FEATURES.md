## 📦 技術實現

### 時間計算邏輯
```typescript
// 轉換時間為分鐘數以便比較
const timeToMinutes = (time: string) => {
  const [hours, minutes] = time.split(':').map(Number);
  return hours * 60 + minutes;
};

// 計算時間差並格式化
const diff = targetMinutes - currentMinutes;
const hours = Math.floor(diff / 60);
const minutes = diff % 60;
const timeStr = hours > 0 ? `${hours}小時${minutes}分鐘` : `${minutes}分鐘`;
```

### 即時更新機制
```typescript
useEffect(() => {
  const timer = setInterval(() => {
    setCurrentTime(new Date());
  }, 1000); // 每秒更新一次
  
  return () => clearInterval(timer);
}, []);
```

### 階段狀態判斷
- **已完成**：當前時間 > 該階段時間 → 灰色 + ✓
- **進行中**：當前時間在該階段 → 高亮 + 動畫脈衝 ●
- **未開始**：當前時間 < 該階段時間 → 淡色 + ○

## 🎯 使用場景

### 場景 1：市集前一天準備
用戶可以在新增市集時設定完整的時間表，包括進場、準備、營業、收攤時間。

### 場景 2：市集當天早上
- 08:00 - 用戶打開應用，看到「距離進場還有 1小時」
- 09:00 - 到達現場，時間軸顯示「進場」階段進行中
- 09:30 - 開始準備攤位，時間軸顯示「準備」階段進行中

### 場景 3：營業時間管理
- 09:45 - 嘗試開始營業，系統提示「距離正式營業時間還有 15分鐘」
- 10:00 - 正式營業時間到，時間軸高亮「營業開始」
- 18:00 - 營業結束，時間軸顯示「營業結束」
- 18:30 - 收攤完成

### 場景 4：遲到情況
- 10:30 - 用戶遲到了，打開應用看到「營業開始」階段已經過去
- 系統仍允許開始營業（因為在營業時間內）
- 時間軸清楚顯示已錯過的階段
