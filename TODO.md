## ⏰ 需求1: 時間管理系統 - 時間軸視覺化（方案C完整版）

**目標：** 提供完整的時間規劃功能，包含進場、準備、營業、收攤時間

### 時間軸結構
```
[進場] ─── [準備] ─── [營業開始] ────────── [營業結束] ─── [收攤]
09:00      09:30      10:00              18:00           18:30
```

### 預設值邏輯
- 營業時間：10:00 - 18:00（必填）
- 準備時間：營業開始前 30 分鐘（可調）
- 進場時間：營業開始前 60 分鐘（可調）
- 收攤時間：營業結束後 30 分鐘（可調）

### 資料結構
```typescript
marketSchedule: {
  entryTime: string;      // 進場時間 "09:00"
  setupTime: string;      // 準備時間 "09:30"
  openTime: string;       // 營業開始 "10:00"
  closeTime: string;      // 營業結束 "18:00"
  packupTime: string;     // 收攤時間 "18:30"
}
```

### 子任務清單

- [ ] **1.1** 更新資料類型定義 (lib/types.ts)
  - 在 `Market` 介面新增 `marketSchedule` 欄位
  - 在事件類型新增對應欄位
  - 定義 `MarketSchedule` 介面

- [ ] **1.2** 更新資料庫 schema (lib/db.ts)
  - 更新資料庫版本號
  - 支援 `marketSchedule` 欄位儲存
  - 處理舊資料遷移（提供預設時間）

- [ ] **1.3** 創建時間軸視覺化組件 (components/TimelineEditor.tsx)
  - 視覺化時間軸 UI
  - 每個時間點可編輯
  - 自動計算時間間隔
  - 驗證時間邏輯（進場 < 準備 < 營業開始 < 營業結束 < 收攤）
  - 提供「使用預設值」按鈕

- [ ] **1.4** 在新增市集頁面整合時間軸編輯器
  - 在日期選擇後新增時間軸區塊
  - 預設載入標準時間
  - 表單提交時包含時間資料

- [ ] **1.5** 在市集編輯表單整合時間軸編輯器
  - 在編輯模態框新增時間軸區塊
  - 載入現有時間設定
  - 更新時同步時間資料

- [ ] **1.6** 在市集詳情頁顯示時間軸資訊
  - 在營業狀態區域顯示今日時間軸
  - 高亮當前時間點
  - 顯示距離下一個時間點的倒數

- [ ] **1.7** 更新營業狀態檢查邏輯 - 考慮時間因素
  - 修改 `canStartSession()` 函數
  - 檢查當前時間是否在營業時間內
  - 提供更精確的提示訊息
