## 3. 殘留產品決策點：保證金顯示

### 3.1 現況

`components/markets/StaffMarketDetailView.tsx` line 442-456 仍渲染保證金金額：

```tsx
{market.deposit && market.deposit > 0 && (
  <div className="flex items-center justify-between px-4 py-3 rounded-xl bg-[#FFF8E7]">
    <div className="flex items-center gap-2">
      <DollarSign className="w-5 h-5 text-[#D4A574]" />
      <div>
        <span className="text-[#3A3A3A]">保證金</span>
        <span className="text-xs text-[#D4A574] ml-2">(需退款)</span>
      </div>
    </div>
    <span className="text-sm font-medium text-[#D4A574]">
      {formatCurrency(market.deposit)}   {/* ← Staff 仍可看到金額 */}
    </span>
  </div>
)}
```

### 3.2 產品決策（2026-06-15 確認）

**選項 A（已採用）**：Staff 可看到保證金**提醒**（名稱 + 需退款標示），但**不看攤位費與租金金額**。

理由：

- 保證金本質上是「市集結束後需退還的款項」，員工需要知道這筆款項存在以避免誤處置（例如不退還給客人、當作收入計入）
- 攤位費與租金是**純粹的成本資訊**，對員工現場工作無關
- 與 C2.30C 「Level 2 員工可見租賃設備存在性（已承租/免費提供/自備），但不可見金額」一致
- 攤位費 / 桌子 / 椅子 / 傘架金額已在 commit `c5cacfa` 改為狀態標籤

### 3.3 殘留技術債（非阻塞）

- 員工 DevTools 仍可看到 `market.deposit` 數值
- 這與 C2.30C 的「UI 層脫敏」原則一致；`lib/permissions/PermissionGate.ts` 並未把 `deposit` 列為 Level 2 敏感欄位
- 若未來產品決策改為「Staff 不可見保證金金額」，需同步更新 PermissionGate `BASE_FIELDS.market` 並改 StaffMarketDetailView
