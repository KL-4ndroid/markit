## 功能現狀

### ✅ 已實作功能

#### 1. 數據記錄層
- **事件類型**: `interaction_recorded`
- **數據結構**:
  ```typescript
  interface InteractionRecordedPayload {
    marketId: string;
    type: InteractionType;  // 按鈕 ID（如 'button_1', 'button_2'）
    productIds?: string[];   // 相關商品 ID
    notes?: string;          // 備註
  }
  ```
- **儲存位置**: IndexedDB events 表

#### 2. 數據收集層
- **QuickInteractionButtons 組件**: 快速記錄互動
- **可自訂按鈕**: 支援 3 個自訂互動按鈕（標籤 + emoji）
- **時間戳記**: 自動記錄互動發生時間

#### 3. 數據分析層
- **互動偏好佔比圖**: 顯示各類互動的比例
- **互動時序熱力圖**: 顯示每小時的互動次數和收入
- **智能洞察**: 自動生成行為分析建議
  - 最頻繁互動類型
  - 轉換率分析（互動 → 成交）
  - 人氣高峰時段

## 應用場景分析

### 🎯 核心價值

#### 1. **了解顧客行為模式**
- **場景**: 攤主想知道顧客最常做什麼互動
- **價值**: 
  - 識別哪些商品最吸引目光（摸摸次數多）
  - 了解顧客猶豫點（詢問次數多但成交少）
  - 優化商品陳列和推銷策略

#### 2. **優化轉換率**
- **場景**: 攤主想提高成交率
- **價值**:
  - 計算「互動 → 成交」轉換率
  - 識別高轉換時段
  - 調整服務策略（如：詢問多時主動介紹）

#### 3. **時段策略優化**
- **場景**: 攤主想知道何時最忙碌
- **價值**:
  - 識別人氣高峰時段
  - 合理安排休息時間
  - 準備充足的商品和零錢

#### 4. **商品關聯分析**
- **場景**: 攤主想知道哪些商品經常一起被關注
- **價值**:
  - 優化商品組合
  - 設計套餐優惠
  - 改善商品陳列位置

## 改進建議

### 🚀 短期改進（1-2 週）

#### 1. **增強互動按鈕的語義化**
**問題**: 目前按鈕 ID 為 `button_1`, `button_2`，不夠直觀

**建議**:
```typescript
// 預設互動類型
const DEFAULT_INTERACTIONS = [
  { id: 'touch', label: '摸摸看', emoji: '👋', color: '#7B9FA6' },
  { id: 'inquiry', label: '詢問', emoji: '💬', color: '#D4A574' },
  { id: 'try', label: '試用', emoji: '✨', color: '#E8A5B8' },
];
```

**優點**:
- 更清晰的數據語義
- 更好的數據分析
- 更容易理解的報表

---

#### 2. **商品關聯記錄優化**
**問題**: 目前 `productIds` 是可選的，很多互動沒有記錄相關商品

**建議**:
- 在快速互動按鈕旁邊增加「選擇商品」功能
- 記錄互動時自動關聯當前查看的商品
- 支援多選商品（顧客同時關注多個商品）

**UI 設計**:
```
┌─────────────────────────────────┐
│ 快速記錄互動                      │
├─────────────────────────────────┤
│ [👋 摸摸看] [💬 詢問] [✨ 試用]   │
│                                 │
│ 相關商品（可選）:                 │
│ [🎨 手工皂] [🧴 護手霜] [+]      │
└─────────────────────────────────┘
```

---

#### 3. **互動記錄快速查看**
**問題**: 目前只能在分析區塊看到統計，無法查看原始記錄

**建議**: 新增「互動記錄列表」
```
┌─────────────────────────────────┐
│ 📝 今日互動記錄 (23 筆)           │
├─────────────────────────────────┤
│ 14:32  👋 摸摸看                 │
│        商品: 手工皂               │
│                                 │
│ 14:28  💬 詢問                   │
│        商品: 護手霜、香氛蠟燭      │
│        備註: 詢問成分             │
│                                 │
│ 14:15  ✨ 試用                   │
│        商品: 護手霜               │
│        → 已成交 ✓                │
└─────────────────────────────────┘
```

### 🔮 長期改進（1-2 個月）

#### 7. **跨市集比較分析**
**價值**: 了解不同市集的顧客行為差異

**功能**:
- 比較不同市集的互動模式
- 識別高轉換率市集的特徵
- 優化市集選擇策略

---

#### 8. **AI 智能建議**
**價值**: 基於歷史數據提供個性化建議

**功能**:
- 「本場市集詢問次數偏高，建議主動介紹商品特色」
- 「14:00-15:00 是人氣高峰，建議提前準備熱門商品」
- 「手工皂的摸摸→成交轉換率達 40%，建議增加庫存」

---

#### 9. **顧客分群分析**
**價值**: 識別不同類型的顧客

**分群**:
- **衝動型**: 摸摸看 → 直接成交（轉換快）
- **猶豫型**: 多次詢問 → 最終成交（需要說服）
- **觀望型**: 只摸摸看，不詢問（需要主動接觸）
- **流失型**: 詢問後離開（需要改進話術）
