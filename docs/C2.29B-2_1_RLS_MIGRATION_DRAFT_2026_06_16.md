## 0. 為何需要 C2.29B-2.1

### 0.1 殘留攻擊面（E2 線上實測確認）

039 修了 view 層、040 修了 view scope，但**底表 RLS 仍放行員工**：

| 攻擊面 | 039 + 040 修法 | 線上實測 |
|---|---|---|
| **#A** Staff 透過底表 RLS SELECT `markets` 拿到 `booth_cost` / `total_profit` | ❌ 沒修 | ✅ **E2 確認**（員工透過 RLS 拉到 `booth_cost=0.00`）|
| **#B** Staff 透過底表 RLS SELECT `products` 拿到 `cost` | ❌ 沒修 | 🟠 預期成立（products `team` 政策從 014 就放行）|
| **#C** Staff 透過底表 RLS SELECT `events` 拿到完整 `payload` | ❌ 沒修 | 🟠 預期成立（events `market_members` 政策從 015 就放行）|

### 0.2 攻擊面根因

**`markets_select_secure`（035）+ `Users can view own and team products`（014）+ `用戶可以查看自己的事件和市集事件`（015）**全部透過 `market_members` 放行員工：

- 員工是市集成員（021 trigger 自動加 `role = 'staff'`）→ 命中 `current_user_market_ids()` → 命中 markets RLS
- 員工透過 `team` 路徑（products 014）拿到 products
- 員工透過 `market_members` 路徑（events 015）拿到 events

→ **員工 SELECT 底表 = 0 row 過濾失敗**。

### 0.3 041 解法

**Owner 判斷採「事實欄位」`m.owner_id` / `p.owner_id` / `e.actor_id`**，不依賴 021 / 002 trigger 維護的 `market_members`。

| 表 | 修法 | 員工結果 |
|---|---|---|
| `markets` | 刪 `markets_select_secure`，新增 `markets_select_owner_only`（`owner_id = auth.uid()`）| 0 row |
| `products` | 刪 `Users can view own and team products`（含 `team` + `is_shared` 路徑），新增 `products_select_owner_only`（`owner_id = auth.uid()`）| 0 row |
| `events` | 刪 2 條線上 SELECT policy（015 + `users_can_view_events`），新增 `events_select_owner_only`（`actor_id = auth.uid() OR market_id IN (current_user_owned_market_ids())`）| 0 row |

## 2. 要收緊的 policies

### 2.1 markets SELECT（1 條 policy → 1 條 policy）

| 現有 policy | 動作 | 取代 |
|---|---|---|
| `markets_select_secure`（035）| 🗑️ DROP | `markets_select_owner_only`（041 新建）|

**Before**（035）：
```sql
CREATE POLICY "markets_select_secure"
ON public.markets FOR SELECT
TO authenticated
USING (
  id IN (SELECT * FROM public.current_user_market_ids())
);
-- 員工是 market_members 一員 → 命中 → 員工可拉完整 markets 表
```

**After**（041）：
```sql
CREATE POLICY "markets_select_owner_only"
ON public.markets FOR SELECT
TO authenticated
USING (
  owner_id = auth.uid()
);
-- 員工 SELECT markets = 0 row
```

### 2.2 products SELECT（1+ 條 policy → 1 條 policy）

| 現有 policy | 動作 | 取代 |
|---|---|---|
| `Users can view own and team products`（014）| 🗑️ DROP | `products_select_owner_only`（041 新建）|
| `products_select_temp`（035，**已 DROP**）| （已不存在）| — |
| 任何其他 staff-friendly SELECT | 🗑️ DROP（DO 區塊）| — |

**Before**（014）：
```sql
CREATE POLICY "Users can view own and team products"
  ON products
  FOR SELECT
  USING (
    owner_id = auth.uid()
    OR
    -- 團隊成員的商品（透過 market_members 關聯）
    owner_id IN (
      SELECT DISTINCT m2.user_id
      FROM market_members m1
      JOIN market_members m2 ON m1.market_id = m2.market_id
      WHERE m1.user_id = auth.uid()
    )
    OR
    -- 共享商品
    is_shared = true
  );
-- team 路徑：員工透過 market_members 拿到完整 products（含 cost）
-- is_shared 路徑：員工透過 is_shared=true 拿到 cost
```

**After**（041）：
```sql
CREATE POLICY "products_select_owner_only"
ON public.products FOR SELECT
TO authenticated
USING (
  owner_id = auth.uid()
);
-- 員工 SELECT products = 0 row
-- is_shared = true 商品：owner 仍可讀，員工改走 view
```

### 2.3 events SELECT（2 條 policy → 1 條 policy）

| 現有 policy | 動作 | 取代 |
|---|---|---|
| `用戶可以查看自己的事件和市集事件`（015）| 🗑️ DROP | `events_select_owner_only`（041 新建）|
| `users_can_view_events`（線上多出，簡化版）| 🗑️ DROP | — |
| 任何其他 staff-friendly SELECT | 🗑️ DROP（DO 區塊）| — |

**Before**（015）：
```sql
CREATE POLICY "用戶可以查看自己的事件和市集事件"
ON events FOR SELECT
TO authenticated
USING (
  actor_id = auth.uid()
  OR
  (
    market_id IS NOT NULL AND
    EXISTS (
      SELECT 1 FROM market_members
      WHERE market_id = events.market_id
        AND user_id = auth.uid()
    )
  )
);
-- 員工透過 market_members 拿到完整 events（含 payload 全部敏感 key）
```

**After**（041）：
```sql
CREATE POLICY "events_select_owner_only"
ON public.events FOR SELECT
TO authenticated
USING (
  -- 自己的事件 (包含 market_id IS NULL 的全局事件)
  actor_id = auth.uid()
  OR
  -- 自己市集的事件 (透過 markets.owner_id 嚴格 owner 判斷)
  market_id IN (SELECT id FROM public.current_user_owned_market_ids())
);
-- 員工 SELECT events = 0 row
```
