# Féria 樣板視覺試點實作紀錄

更新日期：2026-07-15

## 目標

以「具有編輯感與手作氣質的現代市集工作台」為視覺方向，先在三個代表性畫面建立可重複使用的產品語言：

1. 今日首頁：品牌氣質、資訊層級與主要導覽。
2. 市集總覽：成交照片、營運數據與回顧內容的閱讀順序。
3. 現場交易：觸控效率、高對比操作與完成交易後的明確回饋。

## 共用視覺契約

- 頁面以紙白畫布、暖白內容面與墨色文字建立主要結構。
- 霧松綠只負責主要操作，陶土橘、霧藍與乾燥玫瑰負責少量內容分類。
- 一般卡片與控制元件採 8px 圓角；Dialog 維持 12px 圓角。
- 一般區段優先使用留白、細分隔線與全寬內容帶，不把所有資訊都包成浮動卡片。
- 數字使用 tabular numbers；按鈕觸控高度至少 44px，高頻成交操作至少 48–56px。
- 動畫只用於短暫狀態回饋，不使用持續彈跳、放大或高層級浮動特效。

## 三個樣板

### 今日首頁

- 以品牌名稱、今日日期、目前焦點、待處理與近期市集建立由近到遠的閱讀順序。
- 移除大面積漸層與厚重陰影，改用品牌色細線、紙張留白和單一輕陰影。
- 底部導覽改成安靜的暖白工具列，使用陶土色短線標示目前頁面。

### 市集總覽

- 標題、日期、地點與操作按鈕集中於固定的淺色頁首。
- 在營運數據前加入「成交片段」，以既有私有照片、成交金額與支付方式形成內容入口。
- 完整相簿仍保留在成交照片分頁；總覽只顯示精選內容，避免和完整相簿重複。
- 每日表現與工作區導覽統一細邊框、字級、數字節奏與選取狀態。

### 現場交易

- 快速收入與商品銷售使用高對比模式切換，付款方式與數字鍵盤具有穩定的大型觸控區。
- 商品卡、購物車加減、結帳鍵維持固定尺寸，避免內容變動造成版面位移。
- 成交後監聽既有 `deal-closed` 事件，在工作區內顯示金額與支付方式的短暫成功回饋。
- 互動按鈕改為克制的就地確認狀態，不遮住交易畫面。

## 穩定性邊界

本次只調整資訊編排、CSS token、元件呈現與前端操作回饋；不修改：

- 成交資料寫入與離線保存流程。
- 照片拍攝、補拍、上傳與私有圖片存取流程。
- 身分、權限、feature flag 與 route gate。
- 資料庫 schema、R2 設定與 API contract。

市集總覽的照片內容只重用既有 owner album view model、transaction index 與圖片元件，不新增查詢或寫入路徑。

## 驗收重點

- 360px、390px 手機與 1440px 桌面無文字或控制元件重疊。
- 今日首頁可在第一屏辨識目前市集、待處理事項與主要動作。
- 市集總覽沒有照片時不出現空相簿區塊；有照片時可前往完整成交照片分頁。
- 現場交易主要按鈕可單手操作，成交後有可讀但不阻塞下一筆交易的成功提示。
- 原有交易、照片、每日表現與互動功能回歸測試全數通過。
