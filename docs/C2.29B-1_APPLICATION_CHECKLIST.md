## 2. Step 1：開 Supabase SQL Editor

1. 登入 https://supabase.com/dashboard
2. 選擇你的 project
3. 左側 menu → **SQL Editor**
4. 點選 **New query**（空白查詢）
5. 不要選 `postgres` superuser role（預設就是）

## 4. Step 3-6：跑 Section 1-4

**做法**：
- 把 039 草稿 Section 1-4 **整段貼上**（從 `CREATE OR REPLACE FUNCTION public.sanitize_staff_event_payload` 開始，到 Section 4 最後一行 `WHERE ((mm.user_id = auth.uid()) AND (e.actor_id <> auth.uid()));` 結束）

**預期結果**：

- 或 `CREATE FUNCTION` / `CREATE VIEW` 個別成功訊息

**失敗跡象**：
- `ERROR: relation "staff_accessible_markets" does not exist`（view 不存在 → **ROLLBACK**）
- `ERROR: function ... does not exist`（**ROLLBACK**）
- `ERROR: column ... does not exist`（線上 view 結構有變動 → 停止，回報 Cursor）

## 6. Step 8：COMMIT 或 ROLLBACK

### 6.1 全部驗證通過

```sql

```

跑完 COMMIT 後，建議再跑以下 SQL 確認 view 結構正確（`pg_views` 應該顯示更新後的定義）：

```sql
SELECT viewname, length(definition) AS def_length, updated_at
FROM pg_views
WHERE schemaname = 'public'
  AND viewname IN (
    'staff_accessible_markets',
    'staff_accessible_products',
    'staff_accessible_events'
  )
ORDER BY viewname;
```

**預期**：
- 3 個 view 都有
- `def_length` 跟套用前（備份文件）**不同**（因為 039 改寫了 SQL）

### 6.2 任一驗證失敗

```sql

```

跑完 ROLLBACK 後，跑以下 SQL 確認 view 結構回到套用前：

```sql
SELECT viewname, length(definition) AS def_length
FROM pg_views
WHERE schemaname = 'public'
  AND viewname IN (
    'staff_accessible_markets',
    'staff_accessible_products',
    'staff_accessible_events'
  )
ORDER BY viewname;
```

**預期**：
- 3 個 view 都有
- `def_length` 跟套用前（備份文件）**相同**（ROLLBACK 還原成功）

**不要做的事**：
- ❌ ROLLBACK 後不要直接重跑 039（先回報 Cursor 失敗原因）
- ❌ ROLLBACK 後不要用舊 view definition 手動還原（除非 ROLLBACK 沒生效）
- ❌ ROLLBACK 後不要繼續驗證（沒意義）

## 9. 緊急 rollback 流程

如果**套用成功後**才發現問題（不在 transaction 內）：

1. **停止所有前端服務**（避免員工繼續查詢）
2. **到 Supabase SQL Editor**
3. **手動執行** `docs/C2.29_VIEW_BACKUP_2026_06_15.md` §1.1 / §2.1 / §3.1 的 rollback SQL
4. **驗證** pg_views 跟套用前相同
5. **重啟前端服務**
6. **回填事件報告**

**禁止自動 rollback**。
