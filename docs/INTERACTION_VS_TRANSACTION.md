# 互動轉換 vs 金錢轉換 - 設計決策文檔

## 📅 更新時間
2026-02-10

## 🎯 核心問題

在設計互動記錄功能時，我們發現了一個關鍵的概念混淆：

**❌ 錯誤：把「購買」當作互動轉換**  
**✅ 正確：互動轉換應該是「關係往前走了一格」**

---

## 🔍 問題分析

### 產品中存在兩種完全不同的「轉換」

#### ① 行為轉換（Interaction Conversion）
- **發生在**：互動記錄功能
- **是否有金錢**：不一定
- **代表意義**：關係或意圖的完成
- **範例**：加 IG、加 Line、留下聯絡方式

#### ② 金錢轉換（Transaction Conversion）
- **發生在**：快速交易 / 商品交易功能
- **是否有金錢**：一定有
- **代表意義**：營收的完成
- **範例**：實際購買、付款、訂單

### 為什麼不能混淆？

如果把「購買」放在互動記錄的「轉換」中：

1. **數據重複**
   - 互動記錄記了一次「購買」
   - 商品交易又記了一次實際金額
   - 使用者會困惑：「我要記錄兩次嗎？」

2. **語意衝突**
   - 互動記錄的「購買」沒有金額
   - 商品交易的「購買」有金額
   - 同一個詞代表不同意義

3. **分析失準**
   - 轉換率計算會混亂
   - 無法區分「有意願」和「真的買」
   - ROI 分析會出錯

---

## 💡 解決方案

### 一句話定義

> **轉換 = 顧客完成品牌主希望發生的下一步**
> 
> 不是買，不是付錢，是「關係往前走了一格」

### 轉換的通用語意

- ✅ 是「動作」不是狀態
- ✅ 是「非金錢、但可累積價值」的行為
- ✅ 品牌主一看就知道「為什麼要按」
- ❌ 不含金錢語意（買 / 價格 / 訂單 / 成交）

## 📋 更新後的預設配置

### 🍔 食品 / 飲料
```
有興趣：試吃 🍰
有互動：詢問 💬
轉  換：加入追蹤 ➕
```

**為什麼？**
- 食品攤很多是一次性消費
- 追蹤 = 未來回訪的最大價值

### 💍 飾品 / 配件
```
有興趣：拿起 👋
有互動：詢問 💬
轉  換：加 IG 📱
```

**為什麼？**
- 飾品需要建立品牌形象
- IG 是展示作品的最佳平台

### 🎨 插畫 / 創作
```
有興趣：翻看 👀
有互動：聊天 💬
轉  換：加 Line 💚
```

**為什麼？**
- 創作者需要深度溝通
- Line 適合長期聯繫和客製討論

### 👕 服飾
```
有興趣：試穿 👗
有互動：詢問 💬
轉  換：留下聯絡 📝
```

**為什麼？**
- 服飾可能需要訂製或補貨
- 聯絡方式讓後續追蹤更容易

### 📦 其他 / 通用
```
有興趣：看看 👀
有互動：詢問 💬
轉  換：後續聯絡 📞
```

**為什麼？**
- 通用場景，適用於各種攤位
- 強調「建立聯繫」而非「立即成交」

## 🛡️ 防誤解機制

### 在設定精靈中加入明確說明

**位置**：Step 3（理解三個按鈕）的「轉換」卡片底部

**內容**：
```
⚠️ 這裡記錄的是「有價值的互動結果」，
實際銷售金額請使用「商品交易」功能
```

**目的**：
1. 防止使用者重複記錄
2. 防止他質疑數據不準
3. 讓他理解產品是「分工清楚」的

### 更新說明文字

**之前（錯誤）：**
```
顧客完成你想要的行為
不一定是買，但對你有價值

例如：購買 / 加 IG / 加 Line / 訂單
```

**之後（正確）：**
```
顧客完成你想要的下一步
代表關係往前走了一格

例如：加 IG / 加 Line / 留下聯絡 / 加入追蹤

⚠️ 這裡記錄的是「有價值的互動結果」，
實際銷售金額請使用「商品交易」功能
```

## 📊 對數據分析的影響

### 更清晰的漏斗分析

```
有興趣（Interest）
    ↓
有互動（Engage）
    ↓
轉換（Convert）← 建立聯繫
    ↓
[另一個系統]
    ↓
實際購買（Transaction）← 金錢轉換
```

### 可以計算的指標

#### 互動系統（行為轉換）
- 興趣率 = 有興趣 / 總人流
- 互動率 = 有互動 / 有興趣
- 轉換率 = 轉換 / 有互動
- **意義**：衡量「吸引力」和「關係建立能力」

#### 交易系統（金錢轉換）
- 客單價 = 總營收 / 交易次數
- 毛利率 = (營收 - 成本) / 營收
- ROI = (營收 - 投資) / 投資
- **意義**：衡量「獲利能力」

#### 跨系統分析（未來）
- 聯繫轉購率 = 實際購買 / 建立聯繫
- 回購率 = 再次購買 / 首次購買
- 顧客終身價值（LTV）
- **意義**：衡量「長期價值」

## 🎯 設計原則總結

### 三個必須遵守的規則

1. **預設組的「轉換」絕對不能有金錢語意**
   - ❌ 購買、成交、下單、訂單、付款
   - ✅ 加 IG、加 Line、留聯絡、追蹤

2. **轉換必須是「動作」不是「狀態」**
   - ❌ 有興趣、想買、考慮中
   - ✅ 加入、留下、追蹤、聯絡

3. **必須讓使用者理解「分工」**
   - 互動記錄 = 關係建立
   - 商品交易 = 金錢記錄
   - 兩者互補，不重複

## 💭 設計心得

### 為什麼這個問題很重要？

1. **概念清晰度**
   - 混淆概念會讓使用者困惑
   - 清晰的概念讓產品更容易理解

2. **數據準確性**
   - 重複記錄會導致數據失真
   - 分工明確讓分析更準確

3. **產品定位**
   - 這不只是「記帳工具」
   - 這是「關係管理 + 財務管理」的整合系統

### 學到的教訓

1. **不要假設使用者理解你的分類**
   - 必須用明確的語言和範例說明
   - 必須在關鍵位置加入防誤解提示

2. **命名很重要**
   - 「購買」vs「加 IG」傳達完全不同的意圖
   - 好的命名可以引導正確的使用方式

3. **系統分工要清楚**
   - 每個功能有明確的職責
   - 不要讓功能重疊造成困惑

## 📚 參考概念

### 行銷漏斗理論
```
認知（Awareness）
    ↓
興趣（Interest）
    ↓
考慮（Consideration）
    ↓
意圖（Intent）
    ↓
購買（Purchase）
    ↓
忠誠（Loyalty）
```

### 我們的對應
```
[互動系統]
有興趣 ← Awareness + Interest
有互動 ← Consideration + Intent
轉換   ← 建立聯繫（為未來 Purchase 鋪路）

[交易系統]
實際購買 ← Purchase
回購     ← Loyalty
```

## 🎯 一句話總結

> **互動記錄的「轉換」是關係的開始，不是交易的結束**

這個原則確保：
- 使用者不會重複記錄
- 數據分析更準確
- 產品定位更清晰
- 長期價值可追蹤
