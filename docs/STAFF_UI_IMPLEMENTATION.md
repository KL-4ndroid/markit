## 🎨 設計方案

### 配色系統

#### 老闆模式（原有）
- **主色**：`#7B9FA6`（青綠色）
- **次色**：`#D4A574`（金棕色）
- **強調色**：`#E8F3E8`（淺綠）
- **漸變**：`from-[#7B9FA6] to-[#D4A574]`

#### 員工模式（新增）
- **主色**：`#8B7BA6`（紫灰色）
- **次色**：`#A6B4D4`（淺藍紫）
- **強調色**：`#F0E8F3`（淺紫）
- **漸變**：`from-[#8B7BA6] to-[#A6B4D4]`

**設計理念**：
- 紫色系傳達「協作」和「支援」的感覺
- 與青綠色形成對比但不衝突
- 保持專業和優雅的氛圍

## 📦 已創建的文件

### 1. 主題配置系統
**文件**：`lib/theme-config.ts`

提供統一的主題管理函數：
- `getTheme(isStaff)` - 獲取完整主題配色
- `getGradientClass(isStaff)` - 獲取漸變背景類名
- `getPrimaryClass(isStaff)` - 獲取主色調類名
- `getShadowClass(isStaff)` - 獲取陰影類名
- `getBorderClass(isStaff)` - 獲取邊框類名

### 2. 員工專屬組件

#### `components/staff/StaffBadge.tsx`
- 顯示在頂部 Header 的員工身份標籤
- 使用紫色系配色
- 包含圖標和文字

#### `components/staff/SensitiveDataMask.tsx`
- 用於遮罩敏感數據（成本、利潤）
- 支援三種尺寸：sm、md、lg
- 漸變背景 + 鎖定圖標

#### `components/staff/StaffPermissionCard.tsx`
- 顯示員工的權限範圍
- 清晰列出可執行和不可執行的操作
- 使用紫色漸變背景

#### `components/staff/OwnerInfoCard.tsx`
- 顯示員工當前為哪位老闆工作
- 包含皇冠圖標和老闆 Email
- 紫色邊框設計

### 3. Toast 通知主題
**文件**：`lib/toast-theme.ts`

提供角色專屬的通知樣式：
- `staffSuccessToast()` - 員工成功提示（紫色漸變）
- `staffErrorToast()` - 員工錯誤提示（淺紫背景）
- `permissionDeniedToast()` - 權限不足提示
- `roleBasedSuccessToast()` - 根據角色自動選擇樣式

## 📱 使用方式

### 自動檢測角色
所有修改的組件都會自動使用 `useUserRole()` hook 檢測當前用戶角色，無需手動配置。

```tsx
const { isStaff, canViewSensitiveData } = useUserRole();
```

### 應用主題
使用主題配置函數：

```tsx
import { getGradientClass, getShadowClass } from '@/lib/theme-config';

<div className={`${getGradientClass(isStaff)} ${getShadowClass(isStaff)}`}>
  {/* 內容 */}
</div>
```

### 顯示敏感數據遮罩
```tsx
import { SensitiveDataMask } from '@/components/staff/SensitiveDataMask';

{canViewSensitiveData ? (
  <div>顯示實際數據</div>
) : (
  <SensitiveDataMask label="僅老闆可見" />
)}
```

- 使用 Tailwind CSS 的任意值語法：`bg-[#8B7BA6]`

- 單一職責：每個組件只做一件事

- 使用 `useUserRole()` hook 統一管理角色狀態
- 避免重複的角色檢查邏輯
- 條件渲染優化，減少不必要的 DOM 節點
