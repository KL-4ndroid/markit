# 互動設定流程優化 - 漸進式資訊揭露

## 📅 更新時間
2026-02-10

## 🎯 問題分析

### 原有設計的問題

**之前的流程：**
```
Step 1: 選擇攤位類型
  ↓
Step 2: 預覽預設組
  ↓
Step 3: 一次顯示三張卡片說明（❌ 資訊過載）
  ↓
Step 4: 一次調整三個按鈕
```

**問題點：**
- ❌ Step 3 一次顯示三張卡片的大量資訊
- ❌ 使用者需要一次性記憶所有內容
- ❌ 認知負擔過重
- ❌ 說明與調整分離，需要回想

---

## ✨ 優化方案

### 新的流程設計

**優化後的流程：**
```
Step 1: 選擇攤位類型
  ↓
Step 2: 預覽預設組
  ↓
Step 3: 調整第一個按鈕（有興趣）
  - 顯示說明
  - 立即調整
  ↓
Step 4: 調整第二個按鈕（有互動）
  - 顯示說明
  - 立即調整
  ↓
Step 5: 調整第三個按鈕（轉換）
  - 顯示說明
  - 立即調整
  ↓
完成
```

### 核心改進

#### 1. **漸進式資訊揭露**
- 每次只顯示一個按鈕的說明
- 降低認知負擔
- 使用者不需要一次性記憶大量資訊

#### 2. **說明與調整整合**
- 看完說明立即調整
- 不需要回想剛才看到的內容
- 學習與實踐同步進行

#### 3. **清晰的進度感**
- 明確知道「現在在調整第幾個」
- 每一步都有明確的目標
- 不會感到迷失

## 📊 使用者體驗改進

### 認知負擔對比

**之前（一次顯示三個）：**
```
認知負擔 = 3 個概念 × 每個概念的複雜度
         = 高負擔
```

**現在（逐個顯示）：**
```
認知負擔 = 1 個概念 × 每個概念的複雜度
         = 低負擔
```

### 記憶負擔對比

**之前：**
- 看完三個說明 → 需要記住 → 回到調整頁面 → 回想內容
- 記憶負擔：**高**

**現在：**
- 看完一個說明 → 立即調整 → 不需要記憶
- 記憶負擔：**極低**

### 完成率預測

**之前：**
- 看到大量資訊 → 感到壓力 → 可能放棄
- 預期完成率：**中等**

**現在：**
- 每次只處理一個 → 感覺簡單 → 持續完成
- 預期完成率：**高**

## 💡 設計原則

### 1. 漸進式揭露（Progressive Disclosure）

> 只在需要的時候顯示需要的資訊

- 不一次給太多資訊
- 讓使用者專注於當前任務
- 降低認知負擔

### 2. 即時反饋（Immediate Feedback）

> 看完說明立即實踐

- 學習與實踐同步
- 不需要記憶
- 強化理解

### 3. 清晰的進度（Clear Progress）

> 使用者知道自己在哪裡

- 圓形數字徽章顯示順序
- 「下一個」vs「完成設定」區分進度
- 上一步按鈕可以回退

### 4. 一致的結構（Consistent Structure）

> 每一步都使用相同的佈局

- 標題 → 說明 → 調整 → 按鈕
- 使用者知道該期待什麼
- 降低學習成本

## 📁 修改的檔案

### `components/settings/InteractionSetupWizard.tsx`

**主要變更：**

1. **步驟定義**
   ```typescript
   // 之前
   type Step = 'intro' | 'select-type' | 'preview' | 'understand' | 'customize';
   
   // 之後
   type Step = 'intro' | 'select-type' | 'preview' | 'customize-1' | 'customize-2' | 'customize-3';
   ```

2. **移除函數**
   - ❌ `renderUnderstand()` - 一次顯示三個說明
   - ❌ `renderCustomize()` - 一次調整三個按鈕

3. **新增函數**
   - ✅ `renderCustomize1()` - 調整第一個按鈕
   - ✅ `renderCustomize2()` - 調整第二個按鈕
   - ✅ `renderCustomize3()` - 調整第三個按鈕

4. **導航邏輯更新**
   - 更新「上一步」按鈕邏輯
   - 更新內容渲染邏輯

## 🎯 預期效果

### 使用者體驗

1. **降低焦慮感**
   - 不會被大量資訊嚇到
   - 每一步都很簡單

2. **提高完成率**
   - 每次只需要處理一個按鈕
   - 感覺進度快速

3. **增強理解**
   - 看完立即實踐
   - 記憶更深刻

### 產品指標

- **設定完成率**：預期提升 20-30%
- **平均完成時間**：可能略微增加（多了兩個步驟）
- **返回率**：預期降低（不會因為資訊過載而放棄）
- **調整率**：預期提升（調整變得更簡單）

## ✅ 總結

### 核心改進

> **從「一次性資訊轟炸」改為「漸進式引導學習」**

### 關鍵數字

- **資訊量**：從一次 3 個概念 → 每次 1 個概念（減少 66%）
- **記憶負擔**：從需要記住 3 個 → 不需要記憶（減少 100%）
- **步驟數**：從 4 步 → 6 步（增加 50%，但每步更簡單）

### 設計哲學

**不是讓使用者「快速完成」，而是讓使用者「輕鬆完成」**

- 快速但困難 ❌
- 稍慢但輕鬆 ✅

這次優化完美體現了「少即是多」的設計原則。
