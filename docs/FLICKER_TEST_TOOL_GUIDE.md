# 閃爍測試調試工具使用說明

## 🎯 **功能概述**

這是一個專業的動畫參數調試工具，讓您可以實時調整頁面切換動畫的各項參數，找出最佳配置以消除閃爍問題。

---

## 📍 **訪問方式**

### 方法 1：從設定頁面進入（推薦）
1. 點擊底部導航列的「設定」
2. 找到「開發者工具」卡片
3. 點擊「閃爍測試調試工具」按鈕

### 方法 2：直接訪問
在瀏覽器輸入：`/debug/flicker-test`

---

## 🎨 **界面說明**

### 左側：控制面板

#### 1️⃣ **動畫類型選擇**
- **從右滑入**：模擬向右導航（首頁 → 設定）
- **從左滑入**：模擬向左導航（設定 → 首頁）
- **淡入**：模擬子頁面切換

#### 2️⃣ **滑動動畫參數**（僅滑動動畫）

**滑動距離（0-100%）**
- **說明**：內容從側邊滑入的距離
- **建議值**：10-30%
- **效果**：
  - 0%：無滑動效果
  - 20%：輕微滑動（推薦）
  - 50%：明顯滑動
  - 100%：完全從螢幕外滑入

**起始透明度（0-1）**
- **說明**：動畫開始時的透明度
- **建議值**：0.5-0.8
- **效果**：
  - 0：完全透明（可能閃爍）
  - 0.5：半透明（推薦）
  - 0.8：幾乎不透明（最佳）
  - 1：完全不透明（無淡入效果）

**動畫時長（50-500ms）**
- **說明**：動畫持續時間
- **建議值**：150-250ms
- **效果**：
  - 50ms：極快（可能太突兀）
  - 200ms：流暢（推薦）
  - 500ms：緩慢（可能拖沓）

#### 3️⃣ **淡入動畫參數**（僅淡入動畫）

**起始透明度（0-1）**
- **說明**：淡入開始時的透明度
- **建議值**：0.7-0.9
- **效果**：越高越不明顯閃爍

**動畫時長（50-500ms）**
- **說明**：淡入持續時間
- **建議值**：100-150ms
- **效果**：越短越快

#### 4️⃣ **其他設定**

**測試內容**
- **純文字**：簡單文字內容
- **卡片**：卡片網格佈局
- **混合**：包含漸層、卡片、統計等複雜內容

**背景顏色**
- 可以測試不同背景色下的閃爍效果
- 預設：`#FAFAF8`（應用背景色）

---

### 右側：預覽區域

#### 預覽窗口
- 顯示實際的動畫效果
- 背景色會根據設定改變
- 內容會根據選擇的類型顯示

#### 狀態指示器
- **動畫中**：綠色標籤，表示正在播放動畫
- **靜止**：灰色標籤，表示動畫已結束

#### 觀察重點提示
- 動畫開始時是否有白屏閃爍？
- 內容是否平滑出現？
- 是否有跳動或抖動？
- 整體視覺是否流暢？

---

## 🔧 **使用流程**

### Step 1：選擇動畫類型
選擇您想測試的動畫類型（從右滑入/從左滑入/淡入）

### Step 2：調整參數
使用滑桿調整各項參數，觀察右側預覽區域的變化

### Step 3：執行測試
點擊「執行測試」按鈕，觀察動畫效果

### Step 4：重複測試
多次執行測試，確認效果穩定

### Step 5：導出參數
找到最佳配置後，點擊「📋 複製 CSS 參數」

### Step 6：應用到專案
將複製的 CSS 代碼貼到 `app/globals.css` 中

---

## 💡 **最佳實踐**

### 消除閃爍的黃金法則

1. **提高起始透明度**
   - 滑動動畫：≥ 0.5
   - 淡入動畫：≥ 0.8

2. **減少滑動距離**
   - 建議：10-20%
   - 避免：> 50%

3. **縮短動畫時長**
   - 建議：150-250ms
   - 避免：> 300ms

4. **確保背景色一致**
   - 使用與應用相同的背景色
   - 避免白色或透明背景

## 🐛 **常見問題**

### Q1：為什麼調整參數後沒有變化？
**A**：需要點擊「執行測試」按鈕才會應用新參數。

### Q2：如何知道哪個配置最好？
**A**：多次測試，選擇視覺上最流暢、無閃爍的配置。

### Q3：複製的 CSS 要貼到哪裡？
**A**：貼到 `app/globals.css` 文件中，替換現有的動畫定義。

### Q4：可以在手機上測試嗎？
**A**：可以！建議在實際設備上測試，效果更準確。

### Q5：測試計數有什麼用？
**A**：幫助您記錄測試次數，確保充分測試。

## 🎨 **進階技巧**

### 技巧 1：對比測試
1. 記錄當前配置
2. 調整一個參數
3. 對比前後效果
4. 找出最佳值

### 技巧 2：極限測試
1. 先測試極端值（0%, 100%）
2. 觀察最壞和最好情況
3. 在中間找平衡點

### 技巧 3：設備測試
1. 在電腦上找出基本配置
2. 在手機上微調
3. 在平板上驗證
4. 確保所有設備都流暢

### 技巧 4：內容測試
1. 測試純文字（最簡單）
2. 測試卡片（中等複雜）
3. 測試混合內容（最複雜）
4. 確保所有內容都流暢
