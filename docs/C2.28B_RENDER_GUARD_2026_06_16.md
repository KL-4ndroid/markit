## 1. C2.28B 實作細節

### 1.1 統一 render guard 模式

所有 5 個 page 都採用同一個 fail-closed 模式：

```tsx
const { isLoading: isRoleLoading, roleError } = useUserRole();

// ✅ C2.28B：fail-closed render guard（必須在所有 hook 之後）
if (isRoleLoading || roleError) {
  return <RoleLoadingFallback />;
}
```

**設計原則**：

- 必須在**所有 hook 之後**（避免破壞 React hook order 規則）
- 結合 `isRoleLoading || roleError` 兩個條件（loading + error 都是 fail-closed）
- 使用中性 Skeleton `RoleLoadingFallback`（**不顯示**任何 owner-only / staff-only 元素，不預測 isStaff 結果）

### 1.2 `RoleLoadingFallback` 設計（`components/auth/RoleLoadingFallback.tsx`）

```tsx
export function RoleLoadingFallback() {
  return (
    <div className="min-h-screen bg-[#FAFAF8] flex items-center justify-center">
      <div className="max-w-lg mx-auto px-6 w-full">
        <div className="bg-white rounded-[1.5rem] shadow-lg shadow-[#7B9FA6]/10 p-12 text-center">
          <div
            className="w-12 h-12 mx-auto mb-4 border-4 border-[#7B9FA6]/30 border-t-[#7B9FA6] rounded-full animate-spin"
            role="status"
            aria-label="正在載入"
          />
          <p className="text-sm text-[#6B6B6B]">正在確認身分...</p>
        </div>
      </div>
    </div>
  );
}
```

**關鍵設計**：
- ❌ 不顯示 owner-only / staff-only 元素
- ❌ 不洩漏當前 ownerId
- ❌ 不預測 `isStaff` 結果
- ✅ 載入時間通常 < 2 秒，視覺衝擊小
- ✅ 完全符合 C2.30C PermissionGate + C2.28 role fail-closed 的精神

### 1.4 `BottomNavigation.tsx` 守衛（L17-22 + L98 + L122）

```tsx
const { isStaff, isLoading: isRoleLoading, roleError } = useUserRole();

// ✅ C2.28B：fail-closed：loading/error 期間把 analytics 視為禁用（保守處理）
// 即使是 owner，loading 期間也不允許點擊 analytics，避免 race condition
const isRoleUnresolved = isRoleLoading || roleError != null;

// 員工模式下禁用分析功能
if ((isStaff || isRoleUnresolved) && item.id === 'analytics') {
  e.preventDefault();
  toast.error('此功能僅供老闆使用', { ... });
}

// 在 Link 上
const isDisabled = (isStaff || isRoleUnresolved) && item.id === 'analytics';
```

**為什麼要雙重守衛**：
- 員工 session：原本就禁用 analytics
- Owner session 但 role 尚未確認：保守禁用 analytics（避免 race condition）
- Owner session 且 role 確認：可正常點擊
