# 當日流水帳功能

## 功能概述

在市集詳情頁面中，當營業狀態為「營業中」或「已結束」時，會顯示當日流水帳區塊，實時記錄所有交易和互動。

## 顯示條件

### 老闆模式
- ✅ 營業狀態為「營業中」（`operatingPhase === 'operating'`）
- ✅ 營業狀態為「已結束」（`operatingPhase === 'ended'`）

### 員工模式
- ✅ 營業狀態為「營業中」（`operatingStatus.status === 'operating'`）
- ✅ 營業狀態為「已結束」（`operatingStatus.status === 'closed'`）

## 功能特點

### 1. 統計摘要
顯示當日的關鍵指標：
- **成交數**：當日完成的交易筆數
- **互動數**：當日記錄的顧客互動次數
- **收入**：當日總收入金額

### 2. 流水帳列表
按時間倒序顯示所有記錄：

#### 互動記錄
- 顯示互動類型（摸摸、詢問等）
- 顯示時間（HH:mm）
- 使用對應的 emoji 圖標
- 淺色背景，白色邊框

#### 成交記錄
- 顯示交易描述（商品數量、件數）
- 顯示時間（HH:mm）
- 顯示金額
- 使用 💰 圖標
- 淺綠色背景

### 3. 實時更新
自動監聽以下事件：
- `interaction-recorded`：新的互動記錄
- `deal-closed`：新的成交記錄

當有新記錄時，自動重新載入流水帳。

### 4. 空狀態提示
當沒有記錄時，顯示友好的提示：
- 📦 圖標
- "今日尚無交易記錄"
- "開始記錄互動和成交吧！"

## 技術實現

### 組件位置
- `components/markets/DailyTransactionLog.tsx`

### 數據來源
從 IndexedDB 的 `events` 表查詢：
1. **互動記錄**：`type = 'interaction_recorded'`
2. **成交記錄**：`type = 'deal_closed'`

### 日期篩選
```typescript
// 獲取當日 00:00:00 到 23:59:59 的記錄
const startOfDay = new Date(targetDate).setHours(0, 0, 0, 0);
const endOfDay = new Date(targetDate).setHours(23, 59, 59, 999);

// 篩選條件
e.timestamp >= startOfDay && e.timestamp <= endOfDay
```

### 排序邏輯
```typescript
// 按時間倒序（最新的在上面）
logEntries.sort((a, b) => b.timestamp - a.timestamp);
```

## 使用場景

### 場景 1：營業中實時監控
- 員工記錄互動 → 立即顯示在流水帳
- 員工完成交易 → 立即顯示在流水帳
- 老闆可以實時查看當日營業狀況

### 場景 2：營業結束後檢視
- 營業結束後，流水帳仍然顯示
- 可以回顧當日所有交易和互動
- 方便核對收入和統計

### 場景 3：多日市集
- 每天都有獨立的流水帳
- 只顯示當天的記錄
- 不會混淆不同日期的數據

## 與其他功能的整合

### 1. 與互動記錄的整合
當點擊互動按鈕時：
```typescript
<InteractionButtons 
  marketId={marketId}
  onInteractionRecorded={() => {
    window.dispatchEvent(new Event('interaction-recorded'));
  }}
/>
```
觸發 `interaction-recorded` 事件 → 流水帳自動更新

### 2. 與快速交易的整合
當完成交易時：
```typescript
// 在交易完成後觸發事件
window.dispatchEvent(new Event('deal-closed'));
```
觸發 `deal-closed` 事件 → 流水帳自動更新

### 3. 與每日收入統計的區別
- **每日收入統計**：顯示多天市集的每日匯總數據
- **當日流水帳**：顯示單日的詳細交易記錄

## 性能優化

### 1. 數據查詢優化
- 使用 IndexedDB 的索引查詢
- 只查詢當日數據，不載入全部歷史

### 2. 實時更新優化
- 使用事件監聽，而非輪詢
- 只在有新記錄時重新載入

### 3. 列表渲染優化
- 最大高度 400px，超過則滾動
- 避免一次渲染過多記錄
