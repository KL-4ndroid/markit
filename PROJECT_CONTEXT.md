# 📱 Market Pulse - 專案核心上下文 (Project Context)

## 🎯 核心定位與價值 (The Soul)
Market Pulse 是一個專為「市場攤販」設計的數位助手。
- **目標用戶**：在市集、文創攤位現場忙碌的創作者/商販。
- **核心價值**：
  - **Local-First 架構**：本地 Dexie 是唯一資料來源，即使斷網一小時也完全可用。
  - **雲端協作**：Supabase 作為備份和多人協作工具，背景自動同步。
  - **3 秒操作原則**：所有核心動作（記錄交易、互動）必須在 3 秒內單手完成。
  - **數據掌控**：本地資料存於 IndexedDB，確保用戶對營業數據有絕對控制權。

## 🛠️ 技術棧 (Tech Stack)
- **Framework**: Next.js 14+ (App Router, TypeScript)
- **Database**: Dexie.js (IndexedDB Wrapper) - **本地唯一數據源**
- **State**: Zustand (Store)
- **Styling**: Tailwind CSS
- **Icons**: Lucide React (取代所有圖片/Emoji)
- **Analysis**: Recharts (本地數據可視化)

## 🧠 數據架構：Local-First + 事件溯源

### Local-First 原則
**本地 Dexie 是 UI 的唯一資料來源**

```
使用者操作 (UI)
    ↓
寫入 Dexie (Event Sourcing)
    ↓
UI 自動更新 (useLiveQuery)
    ↓
背景同步至 Supabase (非阻塞)
```

### 資料流規則
1. **寫入流程**：UI 操作 → 寫入 Dexie (Event) → 背景同步至 Supabase
2. **讀取流程**：所有讀取操作必須針對 Dexie（使用 `useLiveQuery`）
3. **雲端回流**：Supabase 的新資料必須先寫入 Dexie，再由 Dexie 驅動 UI 更新
4. **斷網處理**：即使斷網一小時，本地操作也必須完全正常，同步邏輯具備自動重試機制

### 事件溯源架構
本專案不直接執行資料覆蓋，而是透過「事件」驅動狀態。
- **邏輯描述**：
  1. 所有行為記錄為不可變的 `events` (例如：`deal_closed`, `market_started`)。
  2. 系統同時維護「快照表」(Snapshots)，如 `markets`, `dailyStats` 以供即時讀取。
- **資料表結構**：
  - `events`: 存儲所有歷史行為（唯一真實來源）。
  - `markets`: 市集快照（從 events 計算得出）。
  - `products`: 商品清單（名稱、價格、分類 - **無圖片**）。
  - `dailyStats`: 每日統計（從 events 計算得出）。
  - `settings`: 用戶本機偏好設定。

## 🚫 嚴格約束條件 (Strict Constraints)

### 禁止事項
1. **❌ 禁止圖片功能**：為了極致效能與儲存空間，**不允許**任何圖片上傳、Base64 儲存或圖片展示功能。所有視覺表達應使用 Lucide Icons 或 CSS 顏色塊。
2. **❌ 禁止直接從 Supabase 讀取資料渲染 UI**：違反 Local-First 原則。
3. **❌ 禁止在主執行緒執行同步操作**：會阻塞 UI，影響用戶體驗。
4. **❌ 禁止混合使用本地和雲端資料源**：必須單一資料來源（Dexie）。

### 必須遵循
1. **✅ 所有讀取使用 `useLiveQuery`**：確保 UI 自動更新。
2. **✅ 所有寫入使用 `recordEvent`**：遵循事件溯源模式。
3. **✅ 雲端同步由 `useSync` 自動處理**：背景非阻塞同步。
4. **✅ 單手操作優化**：所有 UI 元件（尤其是導航與交易按鈕）必須足夠大，適合拇指點擊。
5. **✅ 斷網時完全可用**：本地操作不依賴網路連接。

## 📅 功能模組地圖
1. **市集管理**：新增報名市集資訊為主要功能，首頁需展示未來和目前新增的市集場次，報名狀態流轉（已報名 -> 已錄取 -> 已繳費 -> 如期舉行 以及額外項目[已延期]、[已取消]）。
2. **時間軸管理**：視覺化營業階段（準備、營業中、收攤）。
3. **互動記錄**：單鍵快速記錄「摸摸、詢問、成交」。
4. **交易系統**：極速購物車（點擊商品即加入，選擇支付方式即結帳，也可點擊互動紀錄的[成交]按鈕並跳出金額輸入框，快速紀錄）。
5. **數據分析**：轉換漏斗（互動 -> 成交）與收入/利潤分析。
