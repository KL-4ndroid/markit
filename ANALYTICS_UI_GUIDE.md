## 數據可信度系統

### 可信度等級

| 等級 | 條件 | 圖示 | 說明 |
|------|------|------|------|
| **數據不足** | < 3 場有效市集 | ❌ | 數據蒐集預熱中 |
| **可信度：中** | 3-4 場有效市集 | ⚠️ | 試點診斷模式 |
| **可信度：高** | ≥ 5 場有效市集 | ✅ | 專業建模模式 |

### 有效市集定義
- ✅ 有收入（totalRevenue > 0）
- ✅ 未取消（status ≠ 'cancelled'）
- ✅ 在選定日期範圍內

## 快速模式分析

### 1️⃣ 最有價值市集（前三名）

#### 用戶能看到的資訊
- 🏆 **排名**：第 1/2/3 名
- 📍 **市集名稱**
- 💰 **淨利潤**：實際賺到的錢
- ⏱️ **每小時淨利**：時間效益
- 📈 **回收率**：成本回收倍數
- 🕐 **營業時數**：總工作時間

#### 計算公式

**淨利潤**
```
淨利潤 = 總利潤 - 攤位費 - 報名費 - 設備租金 - 抽成

其中：
- 總利潤 = 總收入 - 商品成本
- 設備租金 = 桌子租金 + 椅子租金 + 傘租金（免費則為 0）
- 抽成 = 總收入 × 抽成比例
```

**每小時淨利**
```
每小時淨利 = 淨利潤 ÷ 總營業時數

總營業時數 = 每日營業時數 × 市集天數
每日營業時數 = (結束時間 - 開始時間) 小時
```

**回收率**
```
回收率 = 總收入 ÷ (攤位費 + 設備租金) × 100%

意義：
- 200% = 收入是成本的 2 倍
- 100% = 剛好回本
- < 100% = 未回本
```

#### 排序規則
1. 優先按「每小時淨利」降序
2. 相同時按「回收率」降序
3. 只顯示淨利潤 ≥ 0 的市集

---

### 2️⃣ 客單價最高市集（前三名）

#### 用戶能看到的資訊
- 🏆 **排名**：第 1/2/3 名
- 📍 **市集名稱**
- 💵 **客單價**：平均每筆交易金額
- 💰 **總收入**
- 🤝 **成交數**

#### 計算公式

**客單價（AOV - Average Order Value）**
```
客單價 = 總收入 ÷ 成交數

意義：
- 反映顧客消費能力
- 數值越高 = 顧客願意花更多錢
- 可能原因：商品定價高、購買多件、高價值商品受歡迎
```

#### 排序規則
- 按「客單價」降序
- 只包含有成交記錄的市集（totalDeals > 0）

---

### 3️⃣ 核心 KPI 卡片

#### 用戶能看到的資訊
- 📊 **平均轉換率**：整體成交效率
- 🔗 **最佳商品組合**：最常一起賣出的商品

#### 計算公式

**平均轉換率**
```
平均轉換率 = 所有市集的平均轉換率

單一市集轉換率 = (成交數 ÷ 互動數) × 100%

意義：
- 反映成交效率
- 數值越高 = 互動後成交的比例越高
```

**商品親和力（最佳組合）**
```
使用 Lift 指標計算商品關聯強度

Lift = P(A,B) ÷ (P(A) × P(B))

其中：
- P(A,B) = 商品 A 和 B 同時出現的機率
- P(A) = 商品 A 出現的機率
- P(B) = 商品 B 出現的機率

Lift > 1 = 正相關（常一起賣）
Lift = 1 = 無關聯
Lift < 1 = 負相關（很少一起賣）
```

### 1️⃣ 市集綜合評分（基礎診斷 - 3場解鎖）

#### 用戶能看到的資訊
- 📊 **平均分數**：所有市集的平均健康評分
- 🏆 **最高分**：表現最好的市集
- 📉 **最低分**：需要改進的市集
- 🎯 **前三名市集**：詳細評分卡片

#### 計算公式

**健康評分（0-100 分）**

**小樣本模式（< 5 場市集）**
```
使用百分位數評分法

1. 計算每個指標的百分位數排名（0-100%）
2. 加權平均：
   健康評分 = (每小時淨利排名 × 40%) + 
              (回收率排名 × 20%) + 
              (轉換率排名 × 20%) + 
              (客單價排名 × 20%)

3. 初期穩定化（3-4 場時）：
   最終評分 = (原始評分 × 70%) + (70 × 30%)
   目的：向平均值靠攏，防止數據波動過大
```

**大樣本模式（≥ 5 場市集）**
```
使用 Z-score 標準化法

1. 計算每個指標的 Z-score：
   Z = (實際值 - 平均值) ÷ 標準差

2. 加權 Z-score：
   加權Z = (每小時淨利Z × 40%) + 
           (回收率Z × 20%) + 
           (轉換率Z × 20%) + 
           (客單價Z × 20%)

3. 轉換為 0-100 分：
   健康評分 = 70 + 加權Z × 15

評分基準：
- Z = 0（平均水平）→ 70 分
- Z = 1（高於平均 1 個標準差）→ 85 分
- Z = -1（低於平均 1 個標準差）→ 55 分
```

**評級標準**
| 分數範圍 | 評級 | 說明 |
|---------|------|------|
| 85-100 | S | 卓越表現 |
| 75-84 | A | 優秀 |
| 60-74 | B | 良好 |
| 45-59 | C | 及格 |
| 0-44 | D | 需改進 |

### 2️⃣ 市集象限分析（深度對比 - 8場解鎖）

#### 用戶能看到的資訊
- ⭐ **明星市集**：高互動 + 高轉換
- 🌱 **潛力市集**：高互動 + 低轉換
- 🎯 **精準市集**：低互動 + 高轉換
- 👀 **觀察市集**：低互動 + 低轉換

#### 計算公式

**象限分類**
```
以平均值為分界線：

1. 計算平均互動數和平均轉換率
2. 分類規則：
   - 明星：互動數 > 平均 AND 轉換率 > 平均
   - 潛力：互動數 > 平均 AND 轉換率 ≤ 平均
   - 精準：互動數 ≤ 平均 AND 轉換率 > 平均
   - 觀察：互動數 ≤ 平均 AND 轉換率 ≤ 平均
```

**互動數計算**
```
互動數 = 詢問數 + 成交數

意義：
- 反映人流量和關注度
- 數值越高 = 越多人對你的商品感興趣
```

**轉換率計算**
```
轉換率 = (成交數 ÷ 互動數) × 100%

意義：
- 反映成交效率
- 數值越高 = 互動後成交的比例越高
```

---

### 3️⃣ 每日收入趨勢圖（深度對比 - 8場解鎖）

#### 用戶能看到的資訊
- 📈 **每日收入折線圖**
- 📅 **日期範圍**：根據篩選條件
- 💰 **收入變化趨勢**

#### 計算公式

**每日收入**
```
1. 遍歷所有市集的成交事件
2. 按日期分組累加收入
3. 生成日期 → 收入的映射

每日收入 = Σ 當日所有成交的總金額
```

### 4️⃣ 商品關聯分析（品牌定位 - 15場解鎖）

#### 用戶能看到的資訊
- 🔗 **商品組合排行**：最常一起賣的商品
- 📊 **關聯強度（Lift）**
- 🎯 **推薦策略**：如何搭配銷售

#### 計算公式

**Lift 指標（關聯強度）**
```
Lift = P(A,B) ÷ (P(A) × P(B))

其中：
- P(A,B) = 商品 A 和 B 同時出現的機率
  = (A 和 B 同時出現的交易數) ÷ (總交易數)

- P(A) = 商品 A 出現的機率
  = (包含 A 的交易數) ÷ (總交易數)

- P(B) = 商品 B 出現的機率
  = (包含 B 的交易數) ÷ (總交易數)

解讀：
- Lift > 1.5：強關聯，建議搭配銷售
- Lift = 1：無關聯
- Lift < 1：負關聯，很少一起賣
```

**支持度（Support）**
```
支持度 = (A 和 B 同時出現的交易數) ÷ (總交易數)

意義：
- 反映組合的普遍性
- 數值越高 = 越多人同時購買這兩個商品
```

**置信度（Confidence）**
```
置信度 = P(B|A) = P(A,B) ÷ P(A)

意義：
- 買了 A 之後買 B 的機率
- 數值越高 = 買 A 的人越可能也買 B
```

---

### 5️⃣ 商品排行榜（無需解鎖）

#### 用戶能看到的資訊
- 🏆 **銷量冠軍**：賣最多的商品
- 💰 **營收冠軍**：賺最多錢的商品
- 📈 **利潤冠軍**：利潤最高的商品

#### 計算公式

**銷量統計**
```
銷量 = Σ 所有交易中該商品的數量

排序：按銷量降序
```

**營收統計**
```
營收 = Σ (售價 × 數量)

排序：按營收降序
```

**利潤統計**
```
利潤 = Σ ((售價 - 成本) × 數量)

排序：按利潤降序
```

## 計算公式總覽

### 核心指標

| 指標 | 公式 | 意義 |
|------|------|------|
| **淨利潤** | 總利潤 - 攤位費 - 報名費 - 設備租金 - 抽成 | 實際獲利 |
| **每小時淨利** | 淨利潤 ÷ 總營業時數 | 時間效益 |
| **回收率** | 總收入 ÷ (攤位費 + 設備租金) × 100% | 成本回收倍數 |
| **客單價** | 總收入 ÷ 成交數 | 平均交易金額 |
| **轉換率** | (成交數 ÷ 互動數) × 100% | 成交效率 |
| **互動數** | 詢問數 + 成交數 | 人流關注度 |

### 進階指標

| 指標 | 公式 | 意義 |
|------|------|------|
| **健康評分** | 加權平均（小樣本）或 Z-score（大樣本） | 綜合表現評分 |
| **Lift** | P(A,B) ÷ (P(A) × P(B)) | 商品關聯強度 |
| **支持度** | 同時出現次數 ÷ 總交易數 | 組合普遍性 |
| **置信度** | P(B\|A) = P(A,B) ÷ P(A) | 條件購買機率 |

## 使用建議

### 新手（< 3 場市集）
1. 專注於記錄數據
2. 觀察「最有價值市集」和「客單價排行」
3. 累積到 3 場後解鎖基礎診斷

### 進階（3-7 場市集）
1. 查看「市集綜合評分」了解整體表現
2. 對比不同市集的優劣勢
3. 根據評分調整策略

### 專業（8+ 場市集）
1. 使用「象限分析」找出市集類型
2. 觀察「趨勢圖」發現規律
3. 15 場後使用「商品關聯分析」優化商品組合

## 常見問題

### Q: 為什麼我的市集沒有出現在排名中？
A: 可能原因：
- 淨利潤為負值（虧損）
- 沒有收入記錄
- 不在選定的日期範圍內
- 市集狀態為「已取消」

### Q: 健康評分是如何計算的？
A: 
- < 5 場：使用百分位數排名法
- ≥ 5 場：使用 Z-score 標準化法
- 3-4 場時會向 70 分靠攏 30% 以穩定評分

### Q: 為什麼有些功能被鎖住了？
A: 需要累積足夠的有效市集數據：
- 基礎診斷：3 場
- 深度對比：8 場
- 品牌定位：15 場

### Q: 什麼是「有效市集」？
A: 同時滿足以下條件：
- 有收入（totalRevenue > 0）
- 未取消（status ≠ 'cancelled'）
- 在選定日期範圍內
