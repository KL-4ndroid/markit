# 測試數據生成器

## 功能描述

在設置頁新增「生成測試數據」按鈕，可快速生成 30 筆完整的測試市集數據，包含：

- ✅ 30 個市集（單日、連續、多選日期）
- ✅ 20 個商品
- ✅ 每個市集的互動記錄（5-15次）
- ✅ 每個市集的成交記錄（2-8筆）

## 新增文件

### `lib/test-data-generator.ts`

測試數據生成核心邏輯，包含：

#### 數據池
- **市集名稱池**：30 個台灣各地市集名稱
- **地點池**：20 個台灣縣市地點
- **商品名稱池**：20 個手作商品名稱
- **商品類別**：handmade, food, accessory, stationery, other
- **商品圖示**：10 個 emoji 圖示
- **顏色池**：6 種柔和色彩

#### 核心函數

1. **`generateTestMarket(index: number)`**
   - 生成單個測試市集
   - 隨機決定日期類型：
     - 40% 單日市集
     - 30% 連續日期（2-3天）
     - 30% 多選日期（3-5天，不連續）
   - 隨機生成時間軸、財務資訊
   - 返回市集 UUID

2. **`generateTestProducts(count: number)`**
   - 生成指定數量的測試商品
   - 隨機價格、成本、庫存
   - 30% 機率為不限庫存
   - 返回商品 UUID 陣列

3. **`generateTestInteractionsAndDeals(marketId, productIds, dates)`**
   - 為每個市集日期生成互動和成交
   - 每天 5-15 次互動（touch, inquiry, interest）
   - 每天 2-8 筆成交
   - 真實的價格分布（低價 30%、中價 40%、高價 30%）
   - 使用 `isBackfill: true` 避免扣庫存

4. **`generateTestData(onProgress?)`**
   - 主函數，生成所有測試數據
   - 支援進度回調
   - 自動設定市集狀態（根據日期）
   - 自動延遲避免過快執行

## 使用方式

1. 進入「設定」頁面
2. 找到「開發者工具」區塊
3. 點擊「生成測試數據（30筆）」按鈕
4. 確認對話框
5. 等待 30-60 秒生成完成
6. 頁面自動刷新

## 生成數據特點

### 市集數據
- **日期分布**：過去 90 天內隨機
- **日期類型**：單日、連續、多選混合
- **時間軸**：隨機生成報到、營業時間
- **財務**：隨機攤位費、租金、抽成
- **狀態**：根據日期智能設定
  - 過去的市集：`completed`（已完成）
  - 進行中的市集：`ongoing`（如期舉行）
  - 未來的市集：`paid`（已繳費）

### 商品數據
- **價格範圍**：50-500 元
- **成本範圍**：20-200 元
- **庫存範圍**：10-100 件（或不限庫存）
- **類別**：手作、食品、飾品、文具、其他

### 互動數據
- **每日互動**：5-15 次
- **類型**：touch, inquiry, interest
- **關聯商品**：50% 機率關聯隨機商品

### 成交數據
- **每日成交**：2-8 筆
- **商品數量**：1-3 個商品
- **每商品數量**：1-3 件
- **價格分布**：更真實的價格區間
  - 30% 低價商品：50-150 元
  - 40% 中價商品：150-350 元
  - 30% 高價商品：350-800 元
- **支付方式**：現金、刷卡、行動支付、其他
- **補登標記**：避免扣庫存

## 技術細節

### 日期生成邏輯

1. **單日市集**
   ```typescript
   dates = [getRandomDate()];
   ```

2. **連續日期**
   ```typescript
   dates = generateContinuousDates(startDate, days);
   // 例如：['2024-02-15', '2024-02-16', '2024-02-17']
   ```

3. **多選日期（不連續）**
   ```typescript
   dates = generateMultipleDates(baseDate, count);
   // 間隔 2-3 天
   // 例如：['2024-02-15', '2024-02-17', '2024-02-20', '2024-02-23']
   ```

### 補登機制

使用 `isBackfill: true` 標記：
- 不扣除商品庫存
- 仍然更新市集統計
- 仍然更新每日統計
- 適合生成歷史測試數據

### 進度回調

```typescript
await generateTestData((current, total, message) => {
  toast.loading(message, {
    description: `${current}/${total} 市集`,
  });
});
```

## 注意事項

1. **數據量**：生成 30 個市集 + 20 個商品 + 數百筆互動/成交
2. **執行時間**：約 30-60 秒
3. **延遲機制**：每個市集間隔 100ms，避免過快
4. **自動刷新**：完成後 2 秒自動刷新頁面
5. **確認對話框**：防止誤操作
