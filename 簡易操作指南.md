# 🎯 超簡單操作指南

## 只需要記住 2 個步驟！

---

## 📝 步驟 1: 在本地開發與測試

### 開始開發
```bash
# 打開終端機，輸入：
npm run dev
```

然後打開瀏覽器訪問：http://localhost:3000

### 開發過程中隨時保存
```bash
# 保存您的更改
git add .
git commit -m "描述您做了什麼"
```

**就這樣！** 繼續開發，想保存就重複上面的命令。

---

## 🚀 步驟 2: 測試完畢，推送到線上

### 方法 A: 推送到測試環境（推薦）
```bash
# 推送到測試環境
git push origin develop
```

- ✅ Vercel 會自動創建一個測試網址
- ✅ 可以在手機或其他設備上測試
- ✅ 不會影響正式網站

### 方法 B: 推送到正式網站
```bash
# 推送到正式網站
npm run deploy:prod
```

- ✅ 自動更新到 https://market-seven-rho.vercel.app/
- ⚠️ 確保功能已測試完成再使用

---

## 🎯 日常工作流程

```
1. 本地開發
   ↓
   npm run dev
   ↓
   (開發、測試、修改)
   ↓
   git add .
   git commit -m "完成某功能"
   ↓

2. 推送到線上測試
   ↓
   git push origin develop
   ↓
   (在 Vercel 測試網址測試)
   ↓

3. 確認沒問題後，推送到正式網站
   ↓
   npm run deploy:prod
   ↓
   完成！✅
```

## 🛠️ 常用命令速查表

| 我想做什麼 | 輸入什麼命令 |
|----------|------------|
| 本地開發 | `npm run dev` |
| 保存更改 | `git add . && git commit -m "描述"` |
| 推送到測試環境 | `git push origin develop` |
| 推送到正式網站 | `npm run deploy:prod` |
| 查看當前狀態 | `git status` |
| 查看所有分支 | `git branch -a` |

## 🚨 遇到問題怎麼辦？

### 問題 1: 推送失敗
```bash
# 先拉取最新代碼
git pull origin develop

# 再推送
git push origin develop
```

### 問題 2: 想撤銷更改
```bash
# 撤銷所有未保存的更改
git checkout .
```

### 問題 3: 不知道現在在哪個分支
```bash
# 查看當前分支
git branch
# 有 * 的就是當前分支
```

### 問題 4: 想切換到開發分支
```bash
# 切換到開發分支
git checkout develop
```
