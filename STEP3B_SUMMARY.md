## 🎯 核心功能

### 狀態流轉流程
```
已報名 → 確認錄取 → 已錄取
       ↓
已錄取 → 確認繳費 → 已繳費
       ↓
已繳費 → 開始營業 → 進行中
       ↓
進行中 → 結束營業 → 已完成
```

### 營業階段
```
進行中狀態下：
- 準備中 ⏸️
- 營業中 ▶️
- 收攤中 ✅
```

### 事件記錄
每次操作都會記錄事件：
- `market_status_changed` - 狀態變更
- `market_started` - 開始營業
- `market_ended` - 結束營業

## 🧪 測試方式

### 1. 測試狀態流轉
1. 在市集列表點擊任一市集
2. 進入詳情頁面
3. 點擊「確認錄取」
4. 查看狀態標籤變為「已錄取」
5. 繼續點擊直到「進行中」
6. 測試營業階段切換
7. 點擊「結束營業」
8. 查看狀態變為「已完成」

### 2. 驗證事件溯源
打開開發者工具：
```
Application → IndexedDB → MarketPulseDB → events
```
應該看到所有狀態變更事件。

### 3. 測試取消與刪除
1. 點擊「取消市集」
2. 確認對話框出現
3. 點擊「確認取消」
4. 查看狀態變為「已取消」
