### 2. 清除本地資料功能

```typescript
const handleClearLocalDatabase = async () => {
  if (!confirm('⚠️ 警告：這將清除所有本地資料...')) {
    return;
  }

  try {
    // 刪除 IndexedDB
    await indexedDB.deleteDatabase('MarketPulseDB');
    
    toast.success('✅ 本地資料庫已清除');
    toast.info('🔄 頁面將在 2 秒後重新載入...');
    
    setTimeout(() => {
      window.location.reload();
    }, 2000);
  } catch (error) {
    console.error('清除本地資料庫失敗:', error);
    toast.error('清除失敗，請稍後再試');
  }
};
```

**特點：**
- ✅ 只清除瀏覽器本地的 IndexedDB
- ✅ 不影響 Supabase 雲端資料
- ✅ 清除後可從雲端重新同步（如已登入）
- ✅ 自動重新載入頁面

### 3. 清除線上資料功能

```typescript
const handleClearOnlineDatabase = async () => {
  if (!user) {
    toast.error('請先登入 Supabase 帳號');
    return;
  }

  const confirmText = prompt(
    '⚠️⚠️⚠️ 超級警告 ⚠️⚠️⚠️\n\n' +
    '這將永久刪除您在 Supabase 雲端的所有資料...\n' +
    '如果您確定要繼續，請輸入「刪除線上資料」：'
  );

  if (confirmText !== '刪除線上資料') {
    toast.info('已取消操作');
    return;
  }

  try {
    toast.loading('正在清除線上資料...');

    // 1. 刪除所有事件
    await supabase.from('events').delete().eq('actor_id', user.id);

    // 2. 刪除所有商品
    await supabase.from('products').delete()...

    // 3. 刪除所有市集
    await supabase.from('markets').delete().eq('owner_id', user.id);

    toast.success('✅ 線上資料已清除');
    toast.info('💡 建議同時清除本地資料以保持一致');
  } catch (error) {
    toast.error(`清除失敗：${error.message}`);
  }
};
```

**特點：**
- ✅ 需要登入才能執行
- ✅ 雙重確認（需要輸入「刪除線上資料」）
- ✅ 刪除 Supabase 雲端的所有資料
- ✅ 影響所有登入此帳號的設備
- ⚠️ 無法恢復

## 📊 功能對比

| 項目 | 清除本地資料 | 清除線上資料 |
|------|-------------|-------------|
| **儲存位置** | 瀏覽器 IndexedDB | Supabase 雲端 |
| **影響範圍** | 僅限當前瀏覽器 | 所有登入的設備 |
| **可恢復性** | ✅ 可從雲端同步 | ❌ 無法恢復 |
| **危險程度** | ⚠️ 中等 | 🚨 極高 |
| **需要登入** | ❌ 不需要 | ✅ 需要 |
| **確認方式** | 簡單確認 | 輸入文字確認 |
| **適用場景** | 資料庫升級、本地損壞 | 重新開始、刪除帳號 |

## 🎯 使用場景

### 清除本地資料

**適用於：**
1. ✅ 資料庫結構升級後出現錯誤
2. ✅ 本地資料損壞或不一致
3. ✅ 想從雲端重新下載最新資料
4. ✅ 清理瀏覽器儲存空間
5. ✅ 切換到新設備前清理舊設備

**操作流程：**
```
1. 點擊「資料庫管理」展開
2. 閱讀「清除本地資料」說明
```
