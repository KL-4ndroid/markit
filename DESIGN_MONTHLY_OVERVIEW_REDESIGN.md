# 📊 首頁「本月概覽」區塊重新設計

## 設計目標

讓使用者清楚地知道統計數據是「本月」的市集、收入、成交額，而不是模糊的「總計」。

---

## 設計改進

### 改進前 ❌

```
┌─────────────────────────────────┐
│                                 │
│  市集場次    總收入    成交數   │
│     5      $12,000     23       │
│                                 │
└─────────────────────────────────┘
```

**問題：**
- 沒有標題，使用者不知道這是什麼統計
- 「市集場次」、「總收入」、「成交數」太模糊
- 不知道是「本月」還是「全部」的統計
- 數字缺乏單位說明

---

### 改進後 ✅

```
┌─────────────────────────────────────────┐
│ 本月概覽              詳情 →            │
│ 2025 年 2 月統計                        │
│ ─────────────────────────────────────── │
│                                         │
│  本月市集    本月收入      本月成交     │
│     5       $12,000         23          │
│   場次      總收入          筆數         │
│                                         │
└─────────────────────────────────────────┘
```

**改進點：**
1. ✅ 添加明確的標題「本月概覽」
2. ✅ 顯示具體的月份「2025 年 2 月統計」
3. ✅ 標籤改為「本月市集」、「本月收入」、「本月成交」
4. ✅ 添加單位說明「場次」、「總收入」、「筆數」
5. ✅ 添加「詳情」按鈕，可跳轉到分析頁面
6. ✅ 使用分隔線區分標題和內容
7. ✅ 數字使用品牌色 (#7B9FA6) 突出顯示
8. ✅ 本月無市集時顯示提示文字

## 實作細節

### 1. 標題區

```tsx
<div className="flex items-center justify-between mb-4 pb-3 border-b border-[#7B9FA6]/10">
  <div>
    <h3 className="text-base font-semibold text-[#3A3A3A] mb-0.5">
      本月概覽
    </h3>
    <p className="text-xs text-[#6B6B6B]">
      {now.getFullYear()} 年 {now.getMonth() + 1} 月統計
    </p>
  </div>
  <button
    onClick={() => router.push('/analytics')}
    className="text-xs text-[#7B9FA6] hover:text-[#6A8E95] transition-colors flex items-center gap-1"
  >
    詳情
    <ArrowRight className="w-3 h-3" />
  </button>
</div>
```

**設計要點：**
- 標題使用 `font-semibold` 強調
- 副標題顯示具體月份
- 添加分隔線區分區塊
- 「詳情」按鈕使用品牌色

### 2. 統計數據區

```tsx
<div className="grid grid-cols-3 gap-4">
  <div className="text-center">
    <div className="text-xs text-[#6B6B6B] mb-1.5">
      本月市集
    </div>
    <div className="text-2xl font-semibold text-[#7B9FA6] tabular-nums mb-0.5">
      {monthlyStats?.marketCount ?? 0}
    </div>
    <div className="text-[10px] text-[#6B6B6B]">場次</div>
  </div>
  
  {/* 其他統計項目... */}
</div>
```

**設計要點：**
- 標籤改為「本月市集」、「本月收入」、「本月成交」
- 數字使用品牌色 `text-[#7B9FA6]` 突出顯示
- 數字使用 `font-semibold` 加粗
- 添加單位說明「場次」、「總收入」、「筆數」
- 使用 `tabular-nums` 確保數字對齊

### 3. 空狀態提示

```tsx
{monthlyStats && monthlyStats.marketCount === 0 && (
  <div className="mt-4 pt-3 border-t border-[#7B9FA6]/10">
    <p className="text-xs text-center text-[#6B6B6B]">
      本月尚未有市集記錄
    </p>
  </div>

```

**設計要點：**
- 當本月無市集時顯示友善提示
- 使用分隔線區分
- 文字居中顯示

## 視覺層級

### 資訊層級（從上到下）

1. **標題層** - 最重要
   - 「本月概覽」- 告訴使用者這是什麼
   - 「2025 年 2 月統計」- 告訴使用者時間範圍

2. **數據層** - 核心內容
   - 數字 - 最大、最醒目（品牌色 + 加粗）
   - 標籤 - 中等大小（「本月市集」等）
   - 單位 - 最小（「場次」、「筆數」等）

3. **操作層** - 次要
   - 「詳情」按鈕 - 提供進一步查看的入口

4. **提示層** - 輔助
   - 空狀態提示 - 只在需要時顯示

## 使用者體驗改進

### 改進前的問題

1. **不清楚時間範圍**
   - 使用者：「這是本月的還是全部的？」
   - 使用者：「市集場次是指什麼？」

2. **缺乏上下文**
   - 使用者：「5 是什麼意思？」
   - 使用者：「總收入是多少錢？」

3. **無法深入了解**
   - 使用者：「我想看更詳細的統計」
   - 沒有提供進一步查看的入口

### 改進後的體驗

1. **清晰的時間範圍** ✅
   - 標題：「本月概覽」
   - 副標題：「2025 年 2 月統計」
   - 標籤：「本月市集」、「本月收入」、「本月成交」

2. **完整的上下文** ✅
   - 數字 + 單位：「5 場次」、「$12,000 總收入」、「23 筆數」
   - 使用者一眼就能理解數據的意義

3. **提供深入入口** ✅
   - 「詳情」按鈕可跳轉到分析頁面
   - 查看更詳細的統計和圖表

4. **友善的空狀態** ✅
   - 本月無市集時顯示提示
   - 避免使用者困惑「為什麼都是 0？」
