# 市集管理功能使用指南

## 🚀 快速開始

### 訪問市集頁面
```
http://localhost:3000/markets
```

或點擊底部導航的「市集」圖標 📅

---

## 📝 新增市集

### 步驟
1. 點擊右上角的 **圓形加號按鈕** ➕
2. 填寫市集資訊：
   - **市集名稱**（必填）：例如「華山文創市集」
   - **地點**（必填）：例如「台北華山文創園區」
   - **日期**（必填）：選擇市集日期
   - **開始時間**：預設 10:00
   - **結束時間**：預設 18:00
   - **報名費**：例如 500
   - **攤位成本**：例如 2000
   - **備註**：其他需要記錄的資訊
3. 點擊 **「建立市集」** 按鈕
4. 看到成功通知 ✅
5. 市集自動出現在列表中

### 注意事項
- 必填欄位會標記紅色星號 *
- 時間欄位有預設值，可以修改
- 費用欄位可以輸入 0
- 備註欄位可選填

## 🏷️ 市集狀態

### 狀態說明
市集有 7 種狀態，每種狀態都有對應的顏色：

| 狀態 | 說明 | 顏色 | Emoji |
|------|------|------|-------|
| 已報名 | 剛建立的市集 | 🟡 柔黃色 | - |
| 已錄取 | 報名通過 | 🟢 柔綠色 | - |
| 已繳費 | 完成繳費 | 🟢 柔綠色 | - |
| 進行中 | 正在營業 | 🔵 霧藍色 | 🎪 |
| 已完成 | 市集結束 | 🌸 柔粉色 | - |
| 已延期 | 市集延期 | 🌸 柔粉色 | - |
| 已取消 | 市集取消 | 🔴 柔粉色+紅字 | - |

### 狀態流轉（Step 3b 將實作）
```
已報名 → 已錄取 → 已繳費 → 進行中 → 已完成
```

## 🔍 篩選市集

### Tab 切換
點擊頂部的 Tab 可以篩選不同狀態的市集：

1. **全部**
   - 顯示所有市集
   - 數量：所有市集總數

2. **進行中**
   - 只顯示正在營業的市集
   - 狀態：ongoing

3. **已報名**
   - 顯示已報名、已錄取、已繳費的市集
   - 狀態：registered, accepted, paid

4. **已結束**
   - 顯示已完成或已取消的市集
   - 狀態：completed, cancelled

### 排序
市集按日期降序排列（最新的在前）

## 📊 市集卡片資訊

### 顯示內容
每張市集卡片顯示：

1. **標題區**
   - 市集名稱
   - 狀態標籤

2. **資訊區**
   - 📅 日期與時間
   - 📍 地點
   - 💰 報名費 / 攤位成本

3. **統計區**（如果有資料）
   - 收入
   - 利潤
   - 成交數

4. **標記**
   - 即將到來 ✨（未來日期且未結束）

### 互動
- **點擊卡片**：查看詳情（Step 3b 將實作）
- **Hover 效果**：陰影提升

## 🗄️ 資料儲存

### 本地儲存
- 所有市集資料存於瀏覽器的 IndexedDB
- 完全離線運作
- 無需網路連線

### 資料結構
每個市集包含：
- 基本資訊（名稱、地點、日期）
- 財務資訊（報名費、攤位成本）
- 統計資訊（收入、利潤、成交數）
- 時間戳（建立時間、更新時間）

### 事件記錄
每次新增市集都會記錄事件：
- 事件類型：`market_created`
- 事件時間：精確到毫秒
- 事件資料：完整的市集資訊

## 🐛 常見問題

### Q: 為什麼點擊市集卡片沒有反應？
A: 市集詳情頁面將在 Step 3b 實作，目前會顯示「即將推出」的提示。

### Q: 可以編輯已建立的市集嗎？
A: 編輯功能將在 Step 3b 實作。

### Q: 可以刪除市集嗎？
A: 刪除功能將在 Step 3b 實作。

### Q: 如何變更市集狀態？
A: 狀態流轉 UI 將在 Step 3b 實作。

### Q: 資料會遺失嗎？
A: 不會。所有資料存於本地 IndexedDB，除非清除瀏覽器資料。

### Q: 可以在不同裝置間同步嗎？
A: 目前不支援。未來計畫加入 Supabase 同步功能。
