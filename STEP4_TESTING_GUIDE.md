# 🚀 Step 4 快速測試指南

## 啟動應用程式

```bash
npm run dev
```

然後在瀏覽器中開啟 `http://localhost:3000`

## 📦 測試商品管理功能

### 1. 進入商品頁面

點擊底部導航的「商品」圖標 📦

### 2. 新增商品

1. 點擊右上角的 **+** 按鈕
2. 填寫商品資訊：
   - **商品名稱**：例如「手工陶杯」
   - **分類**：選擇「手作 🧵」
   - **售價**：例如 500
   - **成本**：例如 200（可選）
   - **庫存**：例如 10（可選）
   - **描述**：例如「純手工製作的陶瓷杯」（可選）
3. 點擊「建立商品」
4. 應該會看到成功提示 ✅

### 3. 測試搜尋功能

1. 在搜尋框輸入關鍵字（例如「陶杯」）
2. 商品列表應該即時篩選

### 4. 測試分類篩選

1. 點擊不同的分類 Tab（手作、食品、飾品等）
2. 商品列表應該只顯示該分類的商品

### 5. 查看商品詳情

1. 點擊任一商品卡片
2. 應該進入商品詳情頁面
3. 查看價格資訊、利潤率、庫存等

### 6. 編輯商品

1. 在商品詳情頁點擊「編輯商品」按鈕
2. 修改商品資訊（例如調整價格）
3. 點擊「儲存變更」
4. 應該會看到更新成功提示 ✅

### 7. 停用/啟用商品

1. 在商品詳情頁點擊「停用商品」按鈕
2. 商品狀態應該變更
3. 返回商品列表，停用的商品不會顯示（因為預設只顯示啟用的商品）

### 8. 刪除商品

1. 在商品詳情頁點擊「刪除商品」按鈕
2. 確認對話框應該彈出
3. 點擊「確認刪除」
4. 商品應該被標記為停用並返回列表頁

## 🔍 檢查事件記錄

### 使用瀏覽器開發者工具

1. 按 F12 開啟開發者工具
2. 切換到「Application」或「儲存空間」標籤
3. 找到「IndexedDB」→「MarketPulseDB」→「events」
4. 應該可以看到所有操作的事件記錄：
   - `product_created`
   - `product_updated`
   - `product_deleted`

### 事件記錄範例

```json
{
  "id": 1,
  "type": "product_created",
  "payload": {
    "name": "手工陶杯",
    "category": "handmade",
    "price": 500,
    "cost": 200,
    "stock": 10,
    "description": "純手工製作的陶瓷杯"
  },
  "timestamp": "2026-01-21T10:30:00.000Z",
  "userId": null
}
```

## 🐛 常見問題

### Q: 商品列表是空的？

A: 請先新增商品。點擊右上角的 + 按鈕建立第一個商品。

### Q: 搜尋沒有結果？

A: 確認搜尋關鍵字是否正確，或清除搜尋框查看所有商品。

### Q: 刪除的商品還在列表中？

A: 刪除是軟刪除，商品會被標記為停用。預設列表只顯示啟用的商品。

### Q: 利潤率沒有顯示？

A: 利潤率需要同時填寫售價和成本才會計算。如果沒有填寫成本，利潤率不會顯示。
