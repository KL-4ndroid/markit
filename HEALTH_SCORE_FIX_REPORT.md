# 健康評分系統修正報告 (v3.2)

## 🎯 問題描述

### 致命盲點：數學天花板問題

在原始的 `calculateMarketHealthScores` 函數中，評分公式存在嚴重的數學設計缺陷：

```typescript
// ❌ 舊版公式
const healthScore = Math.max(0, Math.min(100, 50 + weightedScore * 10));
```

**問題分析：**

統計學上，Z-score 大多落在 **-2 到 +2** 之間（涵蓋 95% 的數據）。

假設一個市集表現極好，各項指標都達到 **PR97** (Z-score ≈ 2)：
- 計算結果：`50 + (2 × 10) = 70 分`
- 評級：**A 級**（需要 80 分才能達到 S 級）

這意味著攤主**幾乎永遠不可能看到 S 級市集**，因為這需要 Z-score 達到 **+3**（在現實數據中極其罕見，僅佔 0.13%）。

### 舊版評級標準的問題

| 評級 | 分數要求 | Z-score 要求 | 百分位數 | 問題 |
|------|----------|--------------|----------|------|
| S 級 | ≥ 80 分 | Z ≥ 3.0 | PR99.87+ | **幾乎不可能達到** |
| A 級 | ≥ 70 分 | Z ≥ 2.0 | PR97.7+ | 極優秀才能達到 |
| B 級 | ≥ 50 分 | Z ≥ 0.0 | PR50+ | 平均表現 |
| C 級 | ≥ 30 分 | Z ≥ -2.0 | PR2.3+ | 較差表現 |
| D 級 | < 30 分 | Z < -2.0 | PR2.3- | 極差表現 |

**核心問題：**
- 平均表現 (Z=0) 只能得到 50 分（B 級），這不符合直覺
- S 級幾乎無法達到，失去激勵作用
- 評分分佈過於集中在 30-70 分之間

## ✅ 修正方案

### 新版評分公式

```typescript
// ✅ 新版公式 (v3.2)
const healthScore = Math.max(0, Math.min(100, 70 + weightedScore * 15));
```

**設計理念：**
1. **基準分提高到 70 分**：平均表現 (Z=0) 得 70 分，處於及格邊緣
2. **乘數擴大到 15**：讓優秀表現更容易達到高分
3. **合理的評級分佈**：讓 S 級可達成但仍具挑戰性

## 📐 數學驗證

### Z-score 分佈特性

在常態分佈中：
- **68%** 的數據落在 Z ∈ [-1, 1] 之間
- **95%** 的數據落在 Z ∈ [-2, 2] 之間
- **99.7%** 的數據落在 Z ∈ [-3, 3] 之間

### 新版公式的覆蓋範圍

```
healthScore = 70 + Z × 15

Z = -2.0 → 70 + (-2.0 × 15) = 40 分 (D 級)
Z = -1.0 → 70 + (-1.0 × 15) = 55 分 (C 級)
Z = 0.0  → 70 + (0.0 × 15)  = 70 分 (B 級)
Z = 1.0  → 70 + (1.0 × 15)  = 85 分 (S 級)
Z = 2.0  → 70 + (2.0 × 15)  = 100 分 (S 級，滿分)
```

**覆蓋範圍：**
- 95% 的市集分數落在 **40-100 分** 之間
- 68% 的市集分數落在 **55-85 分** 之間
- 充分利用 0-100 分的評分空間

## 🔍 實際案例模擬

### 案例 1：頂尖市集

**指標：**
- 每小時淨利：$500（Z = 2.0）
- 攤位費回收率：300%（Z = 2.0）
- 轉換率：80%（Z = 2.0）
- 客單價：$300（Z = 2.0）

**計算：**
```
weightedScore = (2.0 × 0.4) + (2.0 × 0.2) + (2.0 × 0.2) + (2.0 × 0.2)
              = 0.8 + 0.4 + 0.4 + 0.4
              = 2.0

舊版：50 + (2.0 × 10) = 70 分 (A 級) ❌
新版：70 + (2.0 × 15) = 100 分 (S 級) ✅
```

### 案例 2：優秀市集

**指標：**
- 每小時淨利：$300（Z = 1.0）
- 攤位費回收率：200%（Z = 1.0）
- 轉換率：60%（Z = 1.0）
- 客單價：$200（Z = 1.0）

**計算：**

weightedScore = (1.0 × 0.4) + (1.0 × 0.2) + (1.0 × 0.2) + (1.0 × 0.2)

### 案例 4：混合表現市集

**指標：**
- 每小時淨利：$400（Z = 1.5）- 很好

## 📝 程式碼修改

### 修改位置

**檔案：** `lib/analytics-utils.ts`  
**函數：** `calculateMarketHealthScores`  
**行數：** 約 1070-1090 行

### 修改內容

```typescript
// ❌ 舊版
const healthScore = Math.max(0, Math.min(100, 50 + weightedScore * 10));

// 評級
let grade: 'S' | 'A' | 'B' | 'C' | 'D';
if (healthScore >= 80) grade = 'S';
else if (healthScore >= 70) grade = 'A';
else if (healthScore >= 50) grade = 'B';
else if (healthScore >= 30) grade = 'C';
else grade = 'D';
```

```typescript
// ✅ 新版 (v3.2)
// 🔧 修正評分公式：讓平均表現 (Z=0) 得 70 分，優秀表現 (Z=1) 得 85 分
// Z=0 (平均) → 70 分 (及格邊緣)
// Z=1 (優秀) → 85 分 (S 級)
// Z=-1 (稍差) → 55 分 (B/C 級)
// Z=2 (極優) → 100 分 (滿分)
const healthScore = Math.max(0, Math.min(100, 70 + weightedScore * 15));

// 評級（調整後更容易達到 S 級）
let grade: 'S' | 'A' | 'B' | 'C' | 'D';
if (healthScore >= 85) grade = 'S';      // Z ≥ 1.0 (PR84+)
else if (healthScore >= 75) grade = 'A'; // Z ≥ 0.33 (PR63+)
else if (healthScore >= 60) grade = 'B'; // Z ≥ -0.67 (PR25+)
else if (healthScore >= 45) grade = 'C'; // Z ≥ -1.67 (PR5+)
else grade = 'D';                        // Z < -1.67 (PR5-)
```
