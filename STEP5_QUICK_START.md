## 🚀 立即體驗

### 步驟 1：啟動開發伺服器

如果尚未啟動，請執行：

```bash
npm run dev
```

訪問：`http://localhost:3000` 或 `http://localhost:3001`

### 步驟 2：準備測試資料

**需要：**
1. 至少 1 個市集（狀態為「進行中」）
2. 至少 3 個商品（狀態為「啟用」）

**快速建立：**

1. **新增市集**
   - 點擊底部導航「市集」
   - 點擊右上角 + 按鈕
   - 填寫市集資訊
   - 將狀態推進到「進行中」

2. **新增商品**
   - 點擊底部導航「商品」
   - 點擊右上角 + 按鈕
   - 新增 3-5 個不同分類的商品

### 步驟 3：體驗現場操作

1. **進入市集詳情頁**
   - 點擊「市集」
   - 選擇一個「進行中」的市集
   - 進入詳情頁

2. **記錄互動**
   - 看到「快速互動」區域
   - 點擊「詢問 💬」按鈕
   - 觀察 +1 動畫和 Toast 提示
   - 查看「互動次數」增加

3. **完成交易**
   - 點擊「新增交易」按鈕
   - 購物車抽屜開啟
   - 點擊商品卡片加入購物車
   - 選擇支付方式（例如：現金）
   - 點擊「完成結帳」
   - 看到成功提示：「🎉 辛苦了！成交一筆 $XXX」

4. **查看指標**
   - 觀察「即時營業指標」自動更新
   - 成交金額增加
   - 客單價更新
   - 轉換率更新

## 🧪 測試建議

### 基本測試（5 分鐘）

1. **記錄 5 次互動**
   - 詢問 × 2
   - 試吃 × 2
   - 拍照 × 1

2. **完成 2 筆交易**
   - 第一筆：商品 A × 2
   - 第二筆：商品 B × 1

3. **檢查指標**
   - 成交金額正確
   - 客單價正確
   - 轉換率正確

### 進階測試（10 分鐘）

1. **測試購物車**
   - 加入多個商品
   - 調整數量（+/-）
   - 清空購物車
   - 選擇不同支付方式

2. **測試動畫**
   - 觀察 +1 動畫
   - 觀察按鈕縮放
   - 觀察波紋效果

3. **測試邊界情況**
   - 空購物車結帳
   - 快速連點按鈕
   - 關閉購物車

## 📊 指標說明

### 成交金額
- 所有成交的總金額
- 即時累加

### 客單價（AOV）
- Average Order Value
- 計算公式：總金額 / 成交數

### 互動次數
- 所有互動的總和
- 包括：詢問、試吃、拍照、拿名片

### 轉換率
- 成交數 / (互動數 + 成交數) × 100%
- 反映銷售效率

## 🐛 常見問題

### Q: 現場操作區沒有顯示？

**A:** 確認市集狀態是否為「進行中」

### Q: 購物車沒有商品？

**A:** 確認至少有 1 個「啟用」的商品

### Q: 指標沒有更新？

**A:** 刷新頁面或檢查瀏覽器控制台

### Q: +1 動畫沒有顯示？

**A:** 清除瀏覽器快取並重新載入
