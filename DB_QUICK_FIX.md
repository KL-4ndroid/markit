# 🚨 資料庫錯誤快速修復

## 問題

無法進入市集和商品頁面，顯示 `DexieError` 錯誤。

---

## ⚡ 最快修復方法（3 步驟）

### 步驟 1：開啟修復工具

在瀏覽器中訪問：

```
http://localhost:3000/db-fix.html
```

或

```
http://localhost:3001/db-fix.html
```

### 步驟 2：點擊「重置資料庫」

⚠️ **警告：** 這會刪除所有本地資料

### 步驟 3：等待自動重新載入

修復工具會自動：
1. 刪除損壞的資料庫
2. 清除快取
3. 重新載入頁面

---

## 🔧 手動修復方法

如果修復工具無法使用，請手動執行：

### 方法 A：使用瀏覽器控制台

1. **按 F12** 開啟開發者工具

2. **切換到 Console 標籤**

3. **複製並執行以下程式碼：**

```javascript
// 刪除資料庫並重新載入
indexedDB.deleteDatabase('MarketPulseDB');
localStorage.clear();
sessionStorage.clear();
setTimeout(() => location.reload(), 1000);
```

### 方法 B：清除瀏覽器資料

1. **按 F12** 開啟開發者工具

2. **切換到 Application 標籤**

3. **找到 IndexedDB**
   - 展開 `IndexedDB`
   - 找到 `MarketPulseDB`
   - 右鍵點擊 → `Delete database`

4. **清除儲存空間**
   - 點擊左側的 `Storage`
   - 點擊 `Clear site data`

5. **重新載入頁面**
   - 按 `Ctrl + Shift + R`

---

## ✅ 驗證修復

修復成功後，您應該看到：

1. ✅ 首頁正常載入，無錯誤訊息
2. ✅ 可以點擊「市集」進入市集頁面
3. ✅ 可以點擊「商品」進入商品頁面
4. ✅ 控制台無紅色錯誤訊息

---

## 📝 注意事項

### 資料會遺失

重置資料庫會刪除：
- ❌ 所有市集記錄
- ❌ 所有商品資料
- ❌ 所有交易記錄
- ❌ 所有統計資料

### 重新開始

修復後需要：
1. 重新新增市集
2. 重新新增商品
3. 重新記錄交易

---

## 🆘 仍然無法解決？

### 檢查清單

- [ ] 已訪問修復工具頁面
- [ ] 已點擊「重置資料庫」
- [ ] 已重新載入頁面
- [ ] 已清除瀏覽器快取
- [ ] 已關閉其他分頁

### 其他解決方案

1. **重新啟動瀏覽器**
   - 關閉所有瀏覽器視窗
   - 重新開啟瀏覽器

2. **使用無痕模式測試**
   - Chrome: `Ctrl + Shift + N`
   - Firefox: `Ctrl + Shift + P`

3. **更新瀏覽器**
   - 確保使用最新版本

4. **嘗試其他瀏覽器**
   - Chrome
   - Firefox
   - Edge
