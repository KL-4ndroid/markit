## 🎯 核心功能展示

### 市集生命週期
```
1. 建立市集 (Step 3a)
   ↓
2. 狀態：已報名 (registered)
   ↓
3. 確認錄取 → 已錄取 (accepted)
   ↓
4. 確認繳費 → 已繳費 (paid)
   ↓
5. 開始營業 → 進行中 (ongoing)
   ├─ 準備中 (preparation)
   ├─ 營業中 (operating)
   └─ 收攤中 (closing)
   ↓
6. 結束營業 → 已完成 (completed)

特殊情況：
- 取消市集 → 已取消 (cancelled)
- 延期 → 已延期 (postponed)
```

### 事件溯源記錄
```
✅ market_created - 市集建立
✅ market_status_changed - 狀態變更
✅ market_started - 開始營業
✅ market_ended - 結束營業
```

## 💡 技術亮點

### 1. 事件溯源架構
完整實作事件溯源模式：
- 所有操作都記錄為事件
- 事件不可變
- 快照表供快速查詢
- 可從事件重建狀態

### 2. 響應式資料
使用 `useLiveQuery` 實現即時更新：
- 新增市集後列表自動更新
- 狀態變更後 UI 自動更新
- 無需手動刷新

### 3. 動態路由
使用 Next.js App Router 動態路由：
- `/markets` - 列表頁
- `/markets/[id]` - 詳情頁
- 支援任意 ID

### 5. 型別安全

- 減少執行時錯誤

## 🧪 完整測試流程

### 1. 測試市集列表與新增
1. 訪問 http://localhost:3000/markets
2. 查看空狀態
3. 點擊右上角 + 按鈕
4. 填寫市集表單
5. 提交表單
6. 查看成功 Toast
7. 確認市集出現在列表
8. 切換不同 Tab

### 2. 測試市集詳情與狀態流轉
1. 點擊任一市集卡片
2. 進入詳情頁面
3. 查看基本資訊和財務摘要
4. 點擊「確認錄取」
5. 查看狀態變為「已錄取」
6. 繼續點擊直到「進行中」
7. 測試營業階段切換
8. 點擊「結束營業」
9. 查看狀態變為「已完成」

### 3. 測試取消與刪除
1. 建立新市集
2. 進入詳情頁
3. 點擊「取消市集」
4. 確認對話框
5. 點擊「確認取消」
6. 查看狀態變為「已取消」

### 4. 驗證事件溯源
打開開發者工具：
```
Application → IndexedDB → MarketPulseDB
```
- 查看 `events` 表：所有事件記錄
- 查看 `markets` 表：市集快照
