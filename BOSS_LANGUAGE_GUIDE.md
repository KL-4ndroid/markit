# Market Pulse 老闆語翻譯指南

## 🎯 翻譯原則

**技術語言 → 商業語言**
- 不要數學公式，要商業洞察
- 不要統計術語，要行動建議
- 不要抽象概念，要具體數字

---

## 📊 核心指標翻譯

### 1. 健康評分 (Health Score)

#### 技術語言
```
healthScore = 70 + weightedScore × 15
Z-score 標準化
```

#### 老闆語
```
🎯 市集表現評分：85 分（S 級）

這個市集表現優於 84% 的其他市集
建議：值得再來
```

**評級翻譯**：

| 技術 | 老闆語 | 說明 |
|------|--------|------|
| S 級 (≥85) | 🌟 金牌市集 | 表現優於 84% 市集，強烈推薦 |
| A 級 (≥75) | ⭐ 優質市集 | 表現優於 63% 市集，值得再來 |
| B 級 (≥60) | ✅ 穩定市集 | 表現中等偏上，可以考慮 |
| C 級 (≥45) | ⚠️ 待改善市集 | 表現低於平均，需要優化 |
| D 級 (<45) | ❌ 不推薦市集 | 表現墊底，謹慎評估 |

### 2. 轉換率 (Conversion Rate)

#### 技術語言
```
conversionRate = (deals + 1) / (uniqueEngaged + 2)
Laplace 平滑
```

#### 老闆語
```
💰 成交效率：每 10 個客人有 3 個買單

比較：
- 你的市集：30%
- 平均水準：25%
- 結論：✅ 你的銷售話術比別人好
```

### 3. 客單價 (AOV)

#### 技術語言
```
aov = totalRevenue / totalDeals
```

#### 老闆語
```
💵 平均每筆訂單：$450

比較：
- 你的市集：$450
- 平均水準：$380
- 結論：✅ 客人願意花更多錢
```

### 4. 每小時淨利 (Hourly Profit)

#### 技術語言
```
hourlyProfit = netProfit / operatingHours
```

#### 老闆語
```
⏰ 時薪：$120/小時

比較：
- 你的市集：$120/小時
- 平均水準：$80/小時
- 結論：✅ 這個市集很賺
```

### 5. 攤位費回收率 (Booth ROI)

#### 技術語言
```
boothROI = (revenue / totalFixedCost) × 100
```

#### 老闆語
```
🎪 攤位費回收：花 $1000，賺回 $3500

回收率：350%
結論：✅ 攤位費很划算，賺了 3.5 倍
```

### 6. 可信度評分 (Confidence Score)

#### 技術語言
```
confidenceScore = min(interactions/50, deals/20)
```

#### 老闆語
```
📊 數據可靠度：中等

原因：
- 互動數：25 次（建議至少 50 次）
- 成交數：8 筆（建議至少 20 筆）
- 結論：⚠️ 數據量偏少，建議多參加幾次再下結論
```

### 7. Z-score

#### 技術語言
```
Z = (value - mean) / std
Z = 1.5 表示優於平均 1.5 個標準差
```

#### 老闆語
```
📈 表現排名：

你的時薪：$150/小時
市場平均：$100/小時
你的表現：優於 93% 的攤商

結論：🌟 你做得比 93% 的人好
```

### 8. Winsorization

#### 技術語言
```
極端值被拉回到 ±2.5 標準差
```

#### 老闆語
```
🔧 數據修正：

偵測到異常數據（單筆超高價訂單 $99,999）
已自動調整，避免扭曲整體評估

結論：✅ 評分更準確，不受極端值影響
```

## 🏥 診斷類型翻譯

### 1. 流量不足

#### 技術語言
```
interaction < avgInteraction - stdInteraction
```

#### 老闆語
```
🚶 診斷：人流量太少

問題：
- 你的攤位：20 人經過
- 平均水準：50 人經過
- 差距：少了 60%

處方箋：
1. 🎯 換個人多的位置（靠近入口或美食區）
2. 📢 提前在社群宣傳（FB/IG 發文）
3. 🎨 攤位布置更吸睛（大海報、燈光）

預期效果：人流量提升 2-3 倍
```

### 2. 轉換不足

#### 技術語言
```
interaction > avgInteraction && conversionRate < avgConversion
```

#### 老闆語
```
👀 診斷：人很多，但不買單

問題：
- 經過人數：80 人（✅ 很多）
- 成交率：10%（❌ 太低）
- 平均成交率：25%

處方箋：
1. 💬 改善銷售話術（主動介紹、試用）
2. 💰 調整定價策略（組合優惠、限時折扣）
3. 🎁 增加誘因（買一送一、集點卡）

預期效果：成交率提升到 20-25%
```

### 3. 客單價偏低

#### 技術語言
```
deals > avgDeals && aov < avgAOV
```

#### 老闆語
```
🛒 診斷：客人買太少

問題：
- 成交筆數：30 筆（✅ 很多人買）
- 平均客單價：$200（❌ 太低）
- 平均水準：$400

處方箋：
1. 📦 推組合包（買 A 送 B，省 $100）
2. ⬆️ 向上銷售（「要不要加購這個？」）
3. 🎯 設定門檻（滿 $500 送贈品）

預期效果：客單價提升到 $350-400
```

### 4. 精準高效

#### 技術語言
```
interaction < avgInteraction && conversionRate > avgConversion && aov > avgAOV
```

#### 老闆語
```
🎯 診斷：小而美，精準客群

優勢：
- 人流量：30 人（雖然少）
- 成交率：40%（✅ 超高）
- 客單價：$600（✅ 超高）

特點：來的都是精準客戶

處方箋：
1. 📢 擴大曝光（讓更多精準客戶知道你）
2. 📦 增加備貨（避免賣光）
3. 💰 提高定價（客人願意付更多）

預期效果：營收翻倍（人流 × 2）
```

### 5. 均衡穩定

#### 技術語言
```
各項指標接近平均值
```

#### 老闆語
```
⚖️ 診斷：表現穩定，中規中矩

現況：
- 人流量：✅ 正常
- 成交率：✅ 正常
- 客單價：✅ 正常

處方箋：
1. 📊 持續觀察（記錄每次數據）
2. 🔄 小幅優化（測試不同話術）
3. 📈 設定目標（下次提升 10%）

預期效果：維持穩定，逐步成長
```

## 🎯 總結

### 翻譯原則

1. **用數字說話**
   - ❌ Z-score = 1.5
   - ✅ 你比 93% 的人做得好

2. **用比較說明**
   - ❌ conversionRate = 0.3
   - ✅ 每 10 個客人，3 個買單

3. **用行動建議**
   - ❌ 轉換率偏低
   - ✅ 改善銷售話術，預期提升 20%

4. **用視覺化呈現**
   - ❌ 純文字
   - ✅ 圖示 + 顏色 + 進度條

### 關鍵要素

✅ **一眼看懂**
- 大數字、大圖示
- 顏色區分好壞
- 趨勢箭頭

✅ **可執行**
- 具體建議
- 優先順序
- 預期效果

✅ **有對比**
- 你 vs 平均
- 本次 vs 上次
- 目標 vs 現況

老闆不需要懂數學，只需要知道：
1. 這個市集好不好？（評分）
2. 哪裡做得好？（優勢）
3. 哪裡要改進？（處方箋）
4. 怎麼改進？（行動建議）
